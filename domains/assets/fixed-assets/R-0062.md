# Summary

A focused asset management module providing a register view, acquisition, disposal, and depreciation of fixed assets within Ledgius. Operates on the existing GL infrastructure (account, acc_trans, journal) rather than introducing new tables — assets are a contextual lens on capital-account transactions with additional metadata for depreciation tracking.

# Problem / Business Context

Australian SMBs must track fixed assets for:
- ATO instant asset write-off eligibility (threshold changes annually)
- Depreciation calculations (diminishing value per TR 2022/1, prime cost per s40-70 ITAA 1997)
- Capital gains on disposal (CGT event A1)
- BAS capital purchases reporting (GST on capital acquisitions, label G10)
- Balance sheet accuracy (carrying value = cost − accumulated depreciation)

Without a dedicated asset register, users track depreciation in spreadsheets — error-prone, disconnected from the ledger, and invisible to BAS reporting.

# Scope

- Asset register: list of all capital items with cost, category, depreciation method, useful life, and current book value
- Acquisition: record purchase with GL posting (debit asset account, credit bank/AP)
- Disposal: record sale/scrap with gain/loss calculation and GL posting
- Depreciation: calculate and post periodic depreciation entries per asset
- ATO instant write-off: auto-detect eligible assets and fully depreciate in acquisition period
- Vehicle logbook integration: assets of category "Motor Vehicles" link to the existing logbook

# Out of Scope

- Asset revaluation (upward revaluation per AASB 116 — deferred to v2)
- Asset impairment testing (AASB 136)
- Multi-location asset tracking
- Asset maintenance scheduling
- Asset barcode/QR tagging

# Actors / Users

- Business Owner — purchases and disposes of assets
- Bookkeeper — records acquisitions, runs depreciation, reviews register
- Accountant — reviews depreciation schedules, verifies disposal gain/loss

# Preconditions

- Chart of accounts includes asset category accounts (1-2xxx range)
- Tax codes configured (CAP for capital acquisitions)

# Functional Requirements

- **AST-001**: System shall maintain an asset register with: name, category, purchase date, cost, supplier, GL account, depreciation method, useful life, residual value, and current book value.
- **AST-002**: Asset categories shall include: Plant & Equipment, Motor Vehicles, Office Equipment, Furniture & Fittings, IT Equipment. Extensible by tenant.
- **AST-003**: Depreciation methods shall include: Straight Line, Diminishing Value, Instant Write-off. Method selection per asset.
- **AST-004**: Acquisition shall create GL entries: debit the asset's GL account, credit the payment source (bank or AP).
- **AST-005**: Disposal shall calculate gain/loss as (sale proceeds − current book value) and create GL entries: debit bank/AR for proceeds, credit asset account for book value, debit/credit gain or loss on disposal account for the difference.
- **AST-006**: Run Depreciation shall calculate the period's depreciation for all active assets and post journal entries: debit depreciation expense, credit accumulated depreciation.
- **AST-007**: Straight line depreciation: (cost − residual value) / useful life years, prorated to the period.
- **AST-008**: Diminishing value depreciation: book value × (days in period / 365) × (200% / useful life years) per ATO formula.
- **AST-009**: Instant write-off: full cost depreciated in the acquisition period if cost ≤ ATO threshold for the FY.
- **AST-010**: Disposed or fully depreciated assets shall be marked inactive but retained in the register for audit trail.
- **AST-011**: Asset register shall show total cost, total accumulated depreciation, and total current book value.

# Data / Entities / Fields

Leverages existing tables plus a new metadata table:

| Table | Role |
|-------|------|
| account | Asset GL accounts (category A, fixed asset sub-categories) |
| acc_trans | Individual depreciation + acquisition + disposal entries |
| transactions | Parent transaction for each depreciation run / acquisition / disposal |
| **asset_register** (new) | Metadata: name, category, purchase_date, cost, depreciation_method, useful_life_years, residual_value, accumulated_depreciation, status, linked account_id, supplier entity_credit_account_id |

# UX / UI Behaviour

- **AST-012**: Asset Register page displays all assets in a sortable/filterable table.
- **AST-013**: Buy Asset form captures all acquisition details and posts GL on submit.
- **AST-014**: Sell/Dispose form shows current book value, calculates gain/loss in real-time.
- **AST-015**: Depreciation page shows schedule table and a "Run Depreciation" action that posts for the current period.
- **AST-016**: Each page carries an InfoPanel explaining the workflow and ATO context.

# Validation Rules

- Purchase cost must be positive
- Useful life must be ≥ 1 year (except instant write-off)
- Sale proceeds cannot be negative
- Depreciation cannot reduce book value below residual value
- Disposal date must be ≥ purchase date

# Security / Permissions

- Minimum role: Bookkeeper (record acquisitions, run depreciation)
- Disposal requires Bookkeeper or higher
- Register is read-only for Viewer role

# Acceptance Criteria

- [ ] Asset register lists all assets with current book values
- [ ] Buy Asset creates GL entries and adds to register
- [ ] Sell/Dispose calculates gain/loss and creates GL entries
- [ ] Run Depreciation posts correct amounts per method
- [ ] Instant write-off fully depreciates eligible assets
- [ ] Disposed assets marked inactive, retained for audit
