# Summary

Foundation task for the fixed-assets feature — delivers a complete, usable entry point for asset management. After this task merges, a user can:

- Acquire an asset (cash mode) with the correct capital + GST GL postings and audit trail.
- See it on the register with header totals.
- Drill into the per-asset detail page with its projected depreciation schedule and (initially empty) depreciation history.
- Edit non-posting fields with a proper audit row.

Subsequent tasks layer on the rest of the lifecycle:
- **T-0030** — bill-linked acquisition modes (create-new-bill, link-existing-bill). Enhances T-0029's cash-mode entry point.
- **T-0031** — Dispose (sell / scrap / donate / trade-in).
- **T-0032** — Depreciation engine.

## Scope note — this task is split across two implementation PRs

Phase 1 (schema + read paths + non-posting edit + Register + Detail pages) **shipped in the initial T-0029 implementation** (ledgius-db PR #26, ledgius-api PR #57, ledgius-web-app PR #53 — all merged 2026-04-22).

Phase 2 (the Acquire command + POST /assets + BuyAssetPage cash-mode wiring) is delivered by a **stacked follow-up PR bundle**:
- ledgius-api `feat/fixed-assets-acquire` — add `AcquireAsset` service command + `POST /assets` handler + `journal.AssetAcquisitionCashCmd` extension + tests.
- ledgius-web-app `feat/fixed-assets-acquire` — wire BuyAssetPage to `useAcquireAsset`, replace the "coming soon" placeholder.

The spec was previously written as if Phase 2 belonged to T-0030. Review feedback on the sibling loans PR #29 flagged that deferring the core create-command out of the foundation task leaves the feature unusable until the follow-up lands. Same fix: Acquire is now in T-0029.

# Phase 2 — Acquisition (STACKED FOLLOW-UP)

## T-0029.12: AcquireAsset service command

Add to `internal/asset/service.go`:

```go
type AcquireAssetCmd struct {
    TenantID              uuid.UUID
    ActorUserID           uuid.UUID

    Name                  string
    Description           string
    CategoryID            uuid.UUID
    PurchaseDate          time.Time
    CostExGST             decimal.Decimal
    GSTAmount             decimal.Decimal
    GSTApplies            bool
    UsefulLifeYears       *int
    ResidualValue         decimal.Decimal
    DepreciationMethod    string
    BusinessUsePct        *decimal.Decimal  // required for Motor Vehicles

    SupplierEntityID      *int

    // Account overrides (optional — defaults resolved from category):
    CapitalAccountIDOverride               *int
    AccumDepreciationAccountIDOverride     *int
    DepreciationExpenseAccountIDOverride   *int

    // Phase 2 delivers cash mode only. T-0030 adds the bill-linked modes
    // (create-new-bill and link-existing-bill) by extending this command
    // with additional entry-mode parameters.
    BankAccountID         int  // cash mode required; T-0030 makes this optional
}

func (s *Service) AcquireAsset(ctx context.Context, cmd AcquireAssetCmd) (*Asset, error)
```

Implementation:

1. **Validate** per R-0062 §Validation Rules: `cost_ex_gst > 0`; `residual_value >= 0 and <= cost_ex_gst`; `depreciation_method` valid; `useful_life_years >= 1` unless method is `instant_writeoff`; `business_use_pct` in [0,100]; category-is-Motor-Vehicles ⇒ business_use_pct required.
2. **Instant-write-off threshold check**: if `depreciation_method = 'instant_writeoff'`, look up the FY threshold from `instant_writeoff_threshold` for the FY containing `purchase_date`. Reject with `ErrInstantWriteoffExceedsThreshold` if `cost_ex_gst > threshold`.
3. **Period-lock check** (AST-016 / A-0040 I9): reject with `ErrPeriodLocked` if `purchase_date` falls in a locked or BAS-lodged period.
4. **Resolve accounts** per A-0041 §1: explicit override > category defaults > system defaults. Fail with `ErrAccountResolution` if any required account is unresolvable.
5. **Begin tx** via `db.RunInTx`.
6. **INSERT `asset_register`** with:
   - `status = 'active'`
   - `accumulated_depreciation = 0`
   - Resolved account IDs
   - `acquisition_transaction_id = null` (filled in step 8).
7. **Post the acquisition journal** via `journal.Post(ctx, journal.AssetAcquisitionCashCmd{...})` applying A-0041 §2:
   - Dr `capital_account_id` for `cost_ex_gst` (tagged `asset_id`).
   - Dr GST Paid for `gst_amount` (if `gst_applies && gst_amount > 0`) (tagged `asset_id`).
   - Cr `bank_account_id` for `cost_ex_gst + gst_amount` (tagged `asset_id`).
8. **Update the asset row** setting `acquisition_transaction_id` = posted parent transaction id (same tx).
9. **Write audit row**: `audit.Write(entity='asset', entity_id=asset.id, action='acquired', payload={cost_ex_gst, gst_amount, category, method, entry_mode='cash'})` — same tx.
10. **COMMIT**. Return the asset with derived `book_value` populated.

Errors:

- `ErrInvalidAssetInput` — validation failure with field-level detail.
- `ErrInstantWriteoffExceedsThreshold` — cost > FY threshold when method is instant_writeoff.
- `ErrAccountResolution` — required account unresolvable.
- `ErrPeriodLocked` — AST-016.
- `ErrIdempotencyConflict` — duplicate acquisition key.

## T-0029.15: Backend tests

Service-level tests for `AcquireAsset`:

- Happy path cash with GST: asserts GL rows match A-0041 §2 table exactly (3 rows, balanced), asset row has correct status + accumulated=0 + acquisition_transaction_id, audit row written with payload.
- Happy path GST-free (GST_applies=false): row 2 omitted.
- Happy path zero-GST (GST applies but gst_amount=0 — edge case): row 2 omitted.
- Motor Vehicles + business_use_pct persists correctly.
- Instant write-off under threshold: acquisition posts normally; subsequent dep run (future T-0032) will fully depreciate.
- Instant write-off over threshold: `ErrInstantWriteoffExceedsThreshold`.
- Period-lock rejection.
- Missing required fields, invalid method, cost ≤ 0, residual > cost, category mismatch, business_use_pct on non-vehicle.
- Idempotency: double-submit produces one asset.
- **Ledger invariant**: `sum(Dr) == sum(Cr)` for every posted transaction.

Handler tests via `httptest` — 201 happy, 403 missing permission, 402 missing feature, 400 malformed.
