# Summary

Implementation plan for the foundation of fixed assets: the `asset_register` schema, backend list / get / update / non-posting-edit endpoints, and the frontend Asset Register list + per-asset Detail page with activity timeline. This task underpins T-0030/31/32 — no acquisition, disposal, or depreciation work can land until the schema and read paths exist.

# Terms

Uses vocabulary from R-0062 §Data and A-0040 §1 verbatim. New here:

- **Detail page** — `/assets/:id`. Primary hub for per-asset actions. Shows header, summary, schedule, history, timeline.
- **Activity timeline** — audit-log-derived view of every event on an asset (create, edit, dep post, dep reverse, dispose, correct).

# Branch

`feat/fixed-assets-register` on `ledgius-specs`, `ledgius-db`, `ledgius-api`, `ledgius-web-app`. Every file created/modified carries `// Spec references: R-0062, A-0040, A-0041, T-0029.` per the project provenance rule.

# Phase 1 — Schema (ledgius-db)

## T-0029.1: asset_category table

`V1.44__asset_category.sql` (next version number — verify at commit time):

- Columns: `id uuid pk`, `tenant_id uuid fk`, `code text not null`, `name text not null`, `default_capital_account_id bigint fk account`, `default_accum_depr_account_id bigint fk account`, `default_depreciation_expense_account_id bigint fk account`, `active bool default true`, `created_at`, `updated_at`.
- Unique `(tenant_id, code)`.
- Comments explaining each column (per project rule).

Seed the five categories from R-0062 AST-002 via a repeatable migration `R__seed_asset_categories.sql` — idempotent upserts keyed on `(tenant_id, code)`, skipped if already present.

## T-0029.2: asset_register table

`V1.45__asset_register.sql`:

- Columns: `id uuid pk`, `tenant_id uuid fk`, `name text not null`, `description text`, `category_id uuid fk asset_category`, `purchase_date date not null`, `cost_ex_gst numeric(20,2) not null check (cost_ex_gst > 0)`, `gst_amount numeric(20,2) not null default 0`, `useful_life_years int check (useful_life_years >= 1 or depreciation_method = 'instant_writeoff')`, `residual_value numeric(20,2) not null default 0`, `depreciation_method text not null check (depreciation_method in ('straight_line','diminishing_value','instant_writeoff'))`, `accumulated_depreciation numeric(20,2) not null default 0`, `business_use_pct numeric(5,2) check (business_use_pct between 0 and 100)`, `status text not null default 'active' check (status in ('draft','active','fully_depreciated','disposed','archived'))`, `capital_account_id bigint fk account`, `accum_depreciation_account_id bigint fk account`, `depreciation_expense_account_id bigint fk account`, `supplier_entity_id bigint fk entity nullable`, `linked_bill_id bigint fk ap_invoice nullable`, `acquisition_transaction_id bigint fk transactions nullable`, `disposal_transaction_id bigint fk transactions nullable`, `disposal_date date nullable`, `correction_id uuid nullable`, `created_at`, `updated_at`.
- Indexes: `(tenant_id, status)`, `(tenant_id, category_id)`, `(tenant_id, purchase_date)`.
- Check: `residual_value <= cost_ex_gst`.
- Check: `disposal_date is null or disposal_date >= purchase_date`.
- Comments on every column.

## T-0029.3: instant_writeoff_threshold table

`V1.46__instant_writeoff_threshold.sql`:

- Columns: `id serial pk`, `tenant_id uuid fk nullable` (null = global default), `fy_start date not null`, `fy_end date not null`, `threshold_aud numeric(20,2) not null`, `created_at`.
- Unique `(coalesce(tenant_id::text, 'global'), fy_start)`.
- Comments.

Seed FY23/24/25/26 ATO thresholds via `R__seed_instant_writeoff_thresholds.sql` (global rows, tenant_id null).

## T-0029.4: acc_trans asset tags

Augment `acc_trans` (additive only — no column drops) per A-0041 §10:

`V1.47__acc_trans_asset_tags.sql`:

- Add nullable columns: `asset_id uuid`, `deductible boolean`, `run_id uuid`, `correction_id uuid`, `restatement_id uuid`, `reclass_from_bill bigint`.
- Index `(tenant_id, asset_id)` partial `where asset_id is not null`.
- Comments explaining each column's purpose and the layer that consumes it (BAS extractor for `reclass_from_bill`, audit for `correction_id`/`restatement_id`, reporting for `deductible`, period close for `run_id`).

**`bas_label` is intentionally not added** — BAS labels (G1/G10/G11/1A/1B) are derived by the BAS extraction layer (A-0042 forthcoming) from source tax documents and tax codes, not tagged on `acc_trans`. The previous draft of A-0041 included `bas_label`; it was removed in PR review to prevent double-counting on bill-reclassification journals and to accommodate BAS cash-vs-accrual method differences.

Rolled back in the unlikely event via a follow-up `V1.4x__drop_acc_trans_asset_tags.sql` — but per A-0009, prefer forward-only.

# Phase 2 — Backend (ledgius-api)

## T-0029.5: package skeleton

`internal/asset/`:

- `model.go` — GORM models for `Asset`, `AssetCategory`, `InstantWriteoffThreshold`.
- `repository.go` — list with filters, get by id, count-by-status, update-non-posting-fields.
- `service.go` — business commands: `ListAssets`, `GetAsset`, `EditNonPosting`. Acquisition / disposal / depreciation commands live here but are implemented by T-0030/31/32.
- `handler.go` — HTTP handlers.
- `routes.go` — `RegisterRoutes(rg, handler)`.
- `model_test.go`, `repository_test.go`, `service_test.go` — table-driven unit tests per A-0009.

Wire into `cmd/server/main.go` behind `middleware.RequireFeature("fixed_assets")` and `middleware.RequireFunction(...)` per R-0062 §Security.

## T-0029.6: endpoints (part)

See the OpenAPI yaml for request/response detail. This task ships:

- `GET /api/v1/assets` — list with filters: `status`, `category_id`, `purchase_from`, `purchase_to`, `q` (name search), `include_disposed` (default false), `page`, `page_size`. Returns list + header totals.
- `GET /api/v1/assets/:id` — single asset with category, accounts, supplier (joined), derived `book_value` computed field.
- `PATCH /api/v1/assets/:id` — non-posting-field edit (name, description, business_use_pct). Writes audit `edited` with diff.
- `GET /api/v1/assets/:id/activity` — reads `audit_log` rows for `entity_type='asset' AND entity_id=:id`, newest first, paginated.
- `GET /api/v1/assets/categories` — list tenant categories for dropdowns.

Remaining endpoints (create, dispose, depreciation) are owned by T-0030/31/32 but the service layer stubs for them land in this task so the package compiles.

## T-0029.7: tests

- Handler tests using `httptest` — happy path + permission denial + malformed input.
- Ledger invariant — not applicable this task (no GL posting here; that comes in T-0030+).

## T-0029.10: Asset Detail page (new)
