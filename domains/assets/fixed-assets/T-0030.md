# Summary

Deliver the asset acquisition flow end-to-end. Backend: `POST /api/v1/assets` that atomically creates an `asset_register` row and posts the acquisition journal per A-0041 §2 or §3. Frontend: wire up `BuyAssetPage` to the endpoint and support both entry modes — cash/bank and bill-linked.

Depends on T-0029 (schema + package skeleton). No depreciation or disposal in this task.

# Terms

- **Cash acquisition** — A-0041 §2. User pays directly from a bank account; no AP bill.
- **Bill-linked acquisition** — A-0041 §3. Two sub-modes: (a) create a new bill as part of the Buy flow, (b) reclassify an existing bill line to a capital account.
- **Acquisition journal** — the parent `transactions` row of type `asset_acquisition` plus its `acc_trans` children per A-0041 §2/§3.

# Phase 1 — Backend

## T-0030.1: service command

Add to `internal/asset/service.go`:

```go
type AcquireAssetCmd struct {
    TenantID              uuid.UUID
    ActorUserID           uuid.UUID
    Name                  string
    Description           string
    CategoryID            uuid.UUID
    PurchaseDate          time.Time
    CostExGST             decimal.Decimal
    GSTAmount             decimal.Decimal
    GSTApplies            bool
    UsefulLifeYears       *int
    ResidualValue         decimal.Decimal
    DepreciationMethod    string
    BusinessUsePct        *decimal.Decimal
    SupplierEntityID      *int64
    // Entry mode — exactly one must be set:
    BankAccountID         *int64    // cash mode
    LinkedBillID          *int64    // bill-reclassification mode
    NewBillDraft          *BillDraft // create-new-bill mode
}

func (s *Service) AcquireAsset(ctx context.Context, cmd AcquireAssetCmd) (*Asset, error)
```

Implementation outline:

1. Validate: exactly one entry mode; category exists and tenant-scoped; cost > 0; residual ≤ cost; method valid; useful life ≥ 1 unless instant write-off; business_use_pct null unless category is Motor Vehicles.
2. Resolve accounts per A-0041 §1: explicit override > category defaults > system defaults. Fail if any required account is unresolvable.
3. Check instant write-off eligibility: look up `instant_writeoff_threshold` for the FY of `purchase_date`. If method is `instant_writeoff` AND `cost_ex_gst > threshold`, reject (user must switch method). If method is NOT instant_writeoff but cost qualifies, allow — user explicitly chose to spread it.
4. Begin DB transaction.
5. INSERT `asset_register` with status=`active`.
6. For cash mode: call `journal.Post(ctx, journal.AssetAcquisitionCashCmd{...})` which applies A-0041 §2 exactly.
7. For bill-reclassification: validate the bill is in a reclassifiable state and has remaining capital-class lines; call `journal.Post(ctx, journal.AssetAcquisitionReclassifyCmd{...})` which applies A-0041 §3.
8. For create-new-bill: call `bill.Service.CreateBill(ctx, cmd.NewBillDraft)` with the line coded to the capital account directly; set the returned bill_id on the asset row and `acquisition_transaction_id` from the bill's own journal.
9. Set `asset.acquisition_transaction_id` = posted transaction id.
10. `audit.Write(ctx, entity='asset', entity_id=asset.id, action='acquired', actor=ActorUserID, payload={cost, method, entry_mode})`.
11. COMMIT. Return asset.

Errors surface as typed: `ErrInvalidAssetInput`, `ErrAccountResolution`, `ErrInstantWriteoffExceedsThreshold`, `ErrBillNotReclassifiable`. Handler maps to HTTP 400 / 409 appropriately.

## T-0030.2: journal service extension

Extend `internal/journal/` with the three new command types. The journal service is the only code that writes `acc_trans` rows — the asset service hands it a command, it produces the rows per A-0041 §2/§3, and returns the transaction id.

- `AssetAcquisitionCashCmd` — inputs: asset, cost_ex_gst, gst_amount, gst_applies, bank_account_id.
- `AssetAcquisitionReclassifyCmd` — inputs: asset, bill_line_ids, target_capital_account_id.
- `AssetAcquisitionNewBillCmd` is **not** needed — bill service posts the bill journal with the capital account directly; no new journal type.

Each command implementation:
- Produces `acc_trans` rows exactly matching the tables in A-0041 §2/§3.
- Sets the asset tags per A-0041 §10 (`asset_id`, `bas_label=G10`, etc.).
- Returns the `transactions.id`.
