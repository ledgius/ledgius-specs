# Summary

Technical architecture for integrating Basiq bank feeds into Ledgius. Covers
the API integration layer, database schema (all in tenant DB), sync engine,
webhook handling, scheduled jobs, and frontend connection management. The
event journal (`bank_feed_event_log`) records the begin and end of every
live-feed event with status and full error context per R-0049 AC-9.

# Database Schema (Tenant DB)

All tables in this feature live in the **tenant** database (`migrations/tenant/`).
The previous draft of this spec incorrectly classified them as platform-level —
that was an error: `bank_feed_connection.bank_account_id` references
`account(id)`, which is per-tenant.

## Sync engine

### Sync scheduling (replaces fragile time.Ticker)

The previous draft proposed a `time.Ticker` goroutine. That breaks on Fly.io
where `auto_stop_machines = "stop"` and multiple machines may run in
parallel. Replace with a **single-leader scheduler**:

- Cron logic runs only on the machine elected leader via a Postgres advisory
  lock (`pg_try_advisory_lock(BANK_FEED_SCHEDULER_KEY)`)
- Default cadence: every `BANK_FEED_SYNC_INTERVAL` (default `4h`)
- Tick body: `SELECT id FROM bank_feed_connection WHERE status='active' AND (last_sync_at IS NULL OR last_sync_at < now() - interval ? )` then dispatch a goroutine per connection (bounded worker pool)
- If the leader machine stops, the lock is released and another machine acquires it on the next tick
- Set `min_machines_running = 1` in `fly.toml` (already true) to guarantee at least one scheduler is alive

    eventID, err := s.events.Begin(ctx, EventTransactionsFetch, conn.ID, nil)  // begin record

    if err != nil {
        return s.events.Fail(ctx, eventID, err)

### Transaction mapping

Per the DSB Banking standard, each `BankingTransaction` carries
`postingDateTime`, `valueDateTime`, `executionDateTime` (with the invariant
`executionDateTime ≤ valueDateTime ≤ postingDateTime`). Basiq returns these
as `postDate` and `transactionDate`.

```go
func mapBasiqTransaction(bt BasiqTransaction, accountID int) BankTransaction {
    return BankTransaction{
        ExternalID:           bt.ID,                     // Basiq stable ID
        TransDate:            bt.PostDate,               // postingDateTime — canonical settled date
        PostDate:             &bt.TransactionDate,       // executionDateTime — bank-acknowledged date
        Amount:               bt.Amount,                 // signed (negative = debit)
        Description:          bt.Description,
        Reference:            bt.Reference,
        TransType:            inferTransType(bt.Amount),
        Balance:              bt.Balance,
        AccountID:            accountID,
        CounterpartyName:     bt.CounterpartyName,
        Channel:              bt.Channel,
        BankTransactionCode:  bt.SubClass,
        ReconciliationStatus: "imported",
        Source:               "feed",
    }
}
```

### Deduplication

Basiq IDs are stable. Existing unique index `(account_id, external_id)` from
`001_bank_import_tables.sql` enforces dedup:

```sql
INSERT INTO bank_transaction (...)
ON CONFLICT (account_id, external_id) DO NOTHING
```

The number of conflicting rows is captured into
`bank_feed_event_log.transactions_duplicate` for the calling event.

## Webhook handler

### Signature verification (canonical Basiq recipe)

Per https://api.basiq.io/docs/webhooks-security:

- **Headers required**: `webhook-id`, `webhook-timestamp`, `webhook-signature`
- **Signing secret**: configured Fly secret `BASIQ_WEBHOOK_SECRET` of form `whsec_<base64>`. Strip the `whsec_` prefix, base64-decode the remainder for use as the HMAC key.
- **Signed content**: `{webhook-id}.{webhook-timestamp}.{raw-body}` (raw body, byte-for-byte unmodified)
- **Algorithm**: HMAC-SHA256, then base64-encode the digest
- **Header value**: space-separated list of `{version},{signature}` entries (e.g. `v1,Hf...`); strip prefix and compare each
- **Replay window**: reject if `|now - webhook-timestamp| > 5 minutes`
- **IP allowlist**: reject if source IP not in {13.238.192.210, 3.24.252.173, 3.104.17.39}

```go
// Spec references: R-0049, A-0025.
// Authoritative: https://api.basiq.io/docs/webhooks-security
func VerifyBasiqSignature(secret, id, ts, sig string, body []byte, now time.Time) error {
    // 1. Parse timestamp; reject if drift > 5 minutes
    // 2. base64-decode the secret part after whsec_
    // 3. Compute HMAC-SHA256 over id + "." + ts + "." + body
    // 4. base64-encode and constant-time compare against any signature in header
}
```

### Event handling

Basiq webhook events (per https://api.basiq.io/docs/webhooks):

- `connections.created` → fetch accounts, create `bank_feed_connection` rows
- `connections.updated` → reflect status change (e.g. consent revoked → status='disconnected')
- `transactions.created` → trigger immediate sync for the connection (saves a poll cycle)

Each handler writes a `webhook.received` begin/end pair to `bank_feed_event_log`.

## Expiry-check job

Daily cron (single-leader, same advisory lock pattern):

1. Begin `expiry.check` event
2. `SELECT id, consent_expires_at FROM bank_feed_connection WHERE status='active' AND consent_expires_at < now() + interval '7 days'`
3. For each: update status to `expiring`, send email via Resend, log a sub-event
4. End `expiry.check` event with count

# Risks / Trade-offs

| Risk | Mitigation |
|------|-----------|
| Basiq sole-supplier dependency | Abstract behind `BankFeedProvider` interface; the `internal/bankfeed/` package isolates Basiq specifics behind `BasiqClient` |
| CDR consent expires every ≤12 months | Daily expiry-check job + T-7 email + reauthorise flow (AC-6) |
| Basiq rate limits | Bounded worker pool + exponential-backoff retry; document observed limits in `event_log` |
| Multi-machine cron double-fire | Postgres advisory-lock single-leader scheduler |
| Webhook spoofing | Mandatory HMAC-SHA256 verification + IP allowlist + 5-min replay window |
| Cost per tenant exceeds plan margin | Track per-tenant API call counts in event_log for cost analysis |
| Bank coverage gaps | Manual upload fallback (AC-7) |
