# Summary

Technical architecture for integrating Basiq bank feeds into Ledgius. Covers the API integration layer, database schema, sync engine, webhook handling, and frontend connection management.

# Database Schema

## Platform DB — V1.04

```sql
-- Bank feed connections (per tenant, in tenant DB)
CREATE TABLE bank_feed_connection (
    id                  SERIAL PRIMARY KEY,
    bank_account_id     INT NOT NULL REFERENCES account(id),
    provider            TEXT NOT NULL DEFAULT 'basiq',
    provider_user_id    TEXT NOT NULL,
    provider_account_id TEXT NOT NULL,
    institution_name    TEXT NOT NULL,
    account_name        TEXT,
    account_number_mask TEXT,
    bsb                 TEXT,
    status              TEXT NOT NULL DEFAULT 'active'
        CHECK (status IN ('active', 'expiring', 'expired', 'disconnected', 'error')),
    consent_expires_at  TIMESTAMPTZ,
    last_sync_at        TIMESTAMPTZ,
    last_sync_status    TEXT DEFAULT 'pending'
        CHECK (last_sync_status IN ('pending', 'success', 'partial', 'failed')),
    last_sync_error     TEXT,
    transactions_synced INT NOT NULL DEFAULT 0,
    created_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at          TIMESTAMPTZ NOT NULL DEFAULT now()
);

COMMENT ON TABLE bank_feed_connection IS 'Active bank feed connections via Basiq. One row per connected bank account.';
COMMENT ON COLUMN bank_feed_connection.provider_user_id IS 'Basiq user ID for this tenant';
COMMENT ON COLUMN bank_feed_connection.provider_account_id IS 'Basiq account ID for this specific bank account';
COMMENT ON COLUMN bank_feed_connection.account_number_mask IS 'Last 4 digits of account number for display';

CREATE INDEX IF NOT EXISTS idx_bank_feed_status ON bank_feed_connection(status);
CREATE UNIQUE INDEX IF NOT EXISTS idx_bank_feed_provider_account ON bank_feed_connection(provider, provider_account_id);

-- Sync history log
CREATE TABLE bank_feed_sync_log (
    id                  SERIAL PRIMARY KEY,
    connection_id       INT NOT NULL REFERENCES bank_feed_connection(id),
    started_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
    completed_at        TIMESTAMPTZ,
    status              TEXT NOT NULL DEFAULT 'running'
        CHECK (status IN ('running', 'success', 'partial', 'failed')),
    transactions_fetched INT NOT NULL DEFAULT 0,
    transactions_new    INT NOT NULL DEFAULT 0,
    transactions_duplicate INT NOT NULL DEFAULT 0,
    error_message       TEXT,
    sync_from           TIMESTAMPTZ,
    sync_to             TIMESTAMPTZ
);

CREATE INDEX IF NOT EXISTS idx_sync_log_connection ON bank_feed_sync_log(connection_id);
```

## API Endpoints

```
# Connection management (authenticated)
POST   /api/v1/bank-feeds/connect         — Initiate connection (returns Basiq consent URL)
GET    /api/v1/bank-feeds/connections      — List connected accounts
GET    /api/v1/bank-feeds/connections/:id  — Connection detail with sync history
POST   /api/v1/bank-feeds/connections/:id/sync    — Force manual sync
DELETE /api/v1/bank-feeds/connections/:id  — Disconnect account
POST   /api/v1/bank-feeds/connections/:id/reconnect — Re-consent for expired connections

# Account mapping (after connection)
POST   /api/v1/bank-feeds/connections/:id/map     — Map Basiq account to GL account

# Webhook (public, signature verified)
POST   /api/v1/bank-feeds/webhook         — Basiq event notifications
```

### Transaction mapping

```go
func mapBasiqTransaction(bt BasiqTransaction, accountID int) BankTransaction {
    return BankTransaction{
        ExternalID:            bt.ID,
        TransDate:             bt.PostDate,
        PostDate:              &bt.TransactionDate,
        Amount:                bt.Amount,
        Description:           bt.Description,
        Reference:             bt.Reference,
        TransType:             inferTransType(bt.Amount),
        Balance:               bt.Balance,
        AccountID:             accountID,
        CounterpartyName:      bt.CounterpartyName,
        Channel:               bt.Channel,
        BankTransactionCode:   bt.SubClass,
        ReconciliationStatus:  "imported",
        // Source: "feed" — distinguishes from manual upload
    }
}
```

## Webhook Handler

Basiq sends webhooks for:
- `connection.created` — new connection established
- `connection.updated` — consent status change
- `transaction.created` — new transactions available (optional — we poll anyway)

- Amber dot: consent expiring soon

3. Get API key from dashboard

5. Set the API key as a Fly.io secret

# Pricing Considerations

Basiq pricing (approximate):
- Per connection per month: ~$2-5 AUD
- Per API call: ~$0.01-0.05 AUD
- Volume discounts available

This cost should be factored into plan pricing. Bank feeds on Essential ($29/mo) with 3 connections = ~$6-15/mo Basiq cost, leaving healthy margin.

# Risks / Trade-offs

- **Basiq dependency**: single aggregator. Mitigate by abstracting behind a `BankFeedProvider` interface for future multi-provider support
- **CDR consent expiry**: connections expire every 90 days. Must handle re-consent gracefully
- **Rate limiting**: Basiq has API rate limits. Sync scheduling must respect them
- **Data lag**: CDR connections are near-real-time but screen-scrape fallback may have 12-24 hour delay
- **Cost per tenant**: Basiq charges per connection. Must ensure plan pricing covers this
- **Bank coverage**: Basiq covers 170+ institutions but some credit unions may be missing
