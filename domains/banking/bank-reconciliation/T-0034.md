# Overview

Full rewrite of the Bank Reconciliation page based on the Claude
design mockup. The existing ~1,876-line monolithic page is replaced
with a modular component structure matching the mockup's split-pane
layout with progress header, transaction queue, allocator pane,
rules drawer, bulk actions, and contextual help integration.

The mockup source (3,639 lines across 12 JSX files) serves as the
specification. Components are ported from vanilla React/JSX to
TypeScript with Tailwind CSS, integrated with the existing API
hooks and data flow.

# Key Design Decisions

1. **Split-pane layout**: CSS Grid with configurable ratio
   (narrow/balanced/wide). Help drawer compresses the grid when
   docked right.

2. **Optimistic updates**: Approve/undo uses local status overrides.
   The API call fires in background. If undo is clicked within 30s,
   the override is reverted before the API call completes.

3. **Rule preview**: When a rule pattern is entered (either in the
   allocator's "create rule" checkbox or the rules drawer test
   field), matching unallocated transactions in the queue are
   highlighted green.

4. **Density modes**: "comfortable" (62px rows) and "compact"
   (52px rows). Saved to localStorage.

5. **Component isolation**: Each component is self-contained with
   props. No direct API calls inside components — all data flows
   through the page orchestrator via props.

# Risks

| Risk | Mitigation |
|---|---|
| Existing API hooks may not match mockup's data shape | Map/transform in the orchestrator layer, don't change the API |
| RulesDrawer is complex (662 lines in mockup) | Port incrementally — basic list first, then drag-reorder, then inline edit |
| Dark mode CSS may conflict with existing Tailwind theme | Use CSS custom properties as an abstraction layer between components and the theme |
| Keyboard nav may conflict with other pages | Scope keyboard handlers to the reconciliation page only (cleanup on unmount) |
