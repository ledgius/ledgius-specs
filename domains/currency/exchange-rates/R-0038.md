# Summary

Manage currencies and dated exchange rates for multi-currency transaction support. The system stores exchange rates with effective dates and supports rate lookup for a given currency pair and date, as well as currency conversion calculations.

# Problem / Business Context

Businesses that trade internationally or deal in multiple currencies need to record exchange rates and convert amounts between currencies. Ledgius must store dated exchange rates and provide rate lookup so that multi-currency invoices and bills can be correctly valued in the base currency (AUD).

# Scope

- Currency listing (from legacy LedgerSMB schema)
- Exchange rate creation with effective date and source
- Exchange rate listing with currency pair filters
- Rate lookup for a specific currency pair and date (most recent rate on or before the date)
- Currency conversion calculation

# Out of Scope

- Automatic exchange rate feeds (from RBA, xe.com, etc.)
- Multi-currency GL revaluation
- Foreign currency gain/loss calculation
- Currency rounding rules per jurisdiction
- Exchange rate deletion/update

# Actors / Users

- Bookkeeper -- enters exchange rates when dealing with foreign currencies
- System -- looks up rates for multi-currency transaction posting
- Accountant -- reviews exchange rates for period-end revaluation

# Preconditions

- User is authenticated
- Currency records exist (seeded from LedgerSMB schema)

# Functional Requirements

- **FX-001**: System shall list all supported currencies from the currency table
- **FX-002**: System shall store exchange rates with: from_curr, to_curr, rate (decimal 18,8), effective_date, optional source label, and created_at timestamp
- **FX-003**: System shall support creating new exchange rates
- **FX-004**: System shall support listing exchange rates with optional from/to currency filters, ordered by effective_date descending, limited to 50 results
- **FX-005**: System shall support rate lookup returning the most recent rate on or before a given date for a currency pair
- **FX-006**: System shall support currency conversion: multiply amount by rate, returning the same amount if from and to currencies are identical
- **FX-007**: Exchange rate precision shall be 8 decimal places (numeric(18,8))

# Validation Rules

- from_curr and to_curr are required
- Rate is required
- Effective date is required and must be a valid date
- Rate lookup requires both from and to currency codes

# Error Handling

- Missing from/to on lookup returns 400: "from and to currency codes required"
- Invalid date format returns 400: "invalid date"
- No rate found for currency pair returns 404
- Invalid effective_date on create returns 400

# Acceptance Criteria

- [x] Currency listing from legacy schema
- [x] Exchange rate creation with effective date and source
- [x] Exchange rate listing with currency pair filters
- [x] Rate lookup returns most recent rate on or before date
- [x] Currency conversion calculation (same-currency passthrough)
- [x] 8 decimal place precision for rates

# Open Questions

1. **Automatic rate feeds**: No integration with external rate providers (RBA, xe.com).
2. **Rate update/delete**: No mechanism to correct or remove erroneous rates.
3. **Inverse rates**: No automatic inverse rate calculation (must enter both directions).
