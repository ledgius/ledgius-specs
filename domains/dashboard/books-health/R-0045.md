# Summary

A full-page "Books Health" dashboard that answers: "If the accountant or ATO looked at my books right now, what would they flag?" Displays traffic-light status panels across seven health dimensions, with expandable detail metrics and a full financial calendar timeline below.

# Business Context

Bookkeepers and business owners often don't know the state of their books until an accountant reviews them or BAS time arrives. Common pain points:

- Bank accounts not reconciled for weeks — discrepancies grow silently
- Invoices ageing beyond 60-90 days with no follow-up
- Bills going overdue, damaging supplier relationships
- Expenses missing receipts, creating audit risk
- GST coding incomplete, blocking BAS lodgement
- Prior months never formally closed, creating period ambiguity

The Books Health page provides an at-a-glance assessment of all these dimensions, highlighting what needs attention before it becomes a problem.

# Acceptance Criteria

## AC-1: Full-page layout

- Accessible from the sidebar navigation (in both Finance and Tasks modes)
- Route: `/books-health`
- Layout: status panels in a responsive grid at the top, full calendar timeline below
- No navigation away required — everything visible on one page with scrolling

## AC-2: Traffic-light status panels

Seven status panels displayed in a responsive grid (3-4 columns on desktop, 1-2 on mobile). Each panel shows:

- Title and icon
- Traffic-light indicator: green (healthy), amber (needs attention), red (action required)
- One-line summary (e.g. "All accounts reconciled within 7 days")
- Key metric value (e.g. "$12,450 outstanding")
- Expandable detail section (click to expand, see breakdown)

### Panel 1: Bank Reconciliation

Measures how current bank reconciliation is across all accounts.

| Status | Condition |
|--------|-----------|
| Green | All bank accounts reconciled within 7 days |
| Amber | 8-14 days since last reconciliation on any account |
| Red | 15+ days since reconciliation, or account never reconciled |

Detail section shows:
- Each bank account with last reconciled date
- Unmatched transaction count per account
- Variance amount if applicable
- Link to reconciliation page

### Panel 2: Receivables (Aged AR)

Measures the health of outstanding invoices.

| Status | Condition |
|--------|-----------|
| Green | No invoices overdue beyond 60 days |
| Amber | Some invoices in the 60-89 day bucket |
| Red | Any invoices 90+ days overdue |

Detail section shows:
- Aged buckets: Current / 30 days / 60 days / 90+ days with counts and amounts
- Total outstanding amount
- Average days to payment (based on recent history)
- Link to invoices page (filtered to overdue)

### Panel 3: Payables (Aged AP)

Measures outstanding bill payment health.

| Status | Condition |
|--------|-----------|
| Green | No bills overdue |
| Amber | Some bills overdue by less than 7 days |
| Red | Bills overdue by 14+ days |

Detail section shows:
- Aged buckets: Current / 30 days / 60 days / 90+ days with counts and amounts
- Upcoming payments due this week
- Total committed (outstanding payables)
- Link to bills page (filtered to overdue)

### Panel 4: Cash Position

Measures available cash and trend direction.

| Status | Condition |
|--------|-----------|
| Green | Positive cash flow this month, sufficient runway |
| Amber | Flat or slightly declining cash position |
| Red | Negative cash flow or critically low balance |

Detail section shows:
- Current bank balance(s) (from last reconciled position)
- Net cash flow this month (total receipts - total payments)
- Projected end-of-month position (current + expected AR due - expected AP due)
- Simple trend arrow (up / flat / down vs prior month)

### Panel 5: Expense Documentation

Measures receipt and documentation coverage for expenses.

| Status | Condition |
|--------|-----------|
| Green | 95%+ of expenses have receipts/documentation attached |
| Amber | 80-95% documented |
| Red | Less than 80% documented |

Detail section shows:
- Total expense transactions this period
- Count with receipt/attachment vs without
- Count of uncoded transactions (no GL account assigned)
- Link to expenses needing attention

### Panel 6: GST / BAS Readiness

Measures readiness to lodge the current BAS.

| Status | Condition |
|--------|-----------|
| Green | All transactions coded, GST reconciled, ready to lodge |
| Amber | Minor unresolved items (< 5 uncoded transactions) |
| Red | Significant uncoded/unreconciled items, or BAS overdue |

Detail section shows:
- Current reporting period and lodgement deadline
- Total transactions in period: coded vs uncoded count
- GST collected (output tax) vs GST paid (input tax) = net position
- BAS lodgement status for current and prior quarter
- Link to BAS page

### Panel 7: Period Close

Measures month-end close readiness and historical close status.

| Status | Condition |
|--------|-----------|
| Green | All prior months closed, current month on track |
| Amber | Current month open but within acceptable window |
| Red | Multiple prior months remain unclosed |

Detail section shows:
- Checklist: bank reconciled, all journals posted, adjustments reviewed, GST reconciled, period locked
- Prior month status (closed / open)
- Days remaining in current period
- Link to period management

## AC-3: Panel expand/collapse

- Each panel is collapsed by default showing the summary card
- Click to expand and see the detail breakdown
- Multiple panels can be expanded simultaneously
- Expanded state persists during the session (not lost on scroll)

## AC-4: Full calendar timeline

Below the status panels, display the Financial Calendar timeline (from R-0043) in full-page format:
- Same data and view range switcher (7 Days / Month / Quarter)
- Full width — not constrained to a popover
- Richer display: items can show more description text than the popover version
- Done/upcoming/overdue grouping same as popover

## AC-5: Refresh and data currency

- Status panels refresh on page load and every 60 seconds
- A "last updated" timestamp is visible
- Manual refresh button available
- All data comes from the same API sources used by the financial calendar plus new health-check queries

## AC-6: Actionable links

Every status panel links to the relevant page where the user can take action:
- Bank Reconciliation → `/banking`
- Receivables → `/invoices` (filtered to overdue)
- Payables → `/bills` (filtered to overdue)
- Cash Position → `/reports` (cash flow)
- Expense Documentation → expenses/receipts view
- GST / BAS → `/bas`
- Period Close → period management

## AC-7: Role-based visibility

All roles can view the Books Health page. The data shown is always for the current tenant.
