# Summary

A clickable date in the application header that opens a popover timeline showing completed and upcoming financial tasks for the current tenant. Provides at-a-glance visibility of what the bookkeeper has done and what is coming up, replacing the need for separate task management or phone/email coordination with the accountant.

# Business Context

Bookkeepers and business owners need to know: "What have I done? What's coming up? What's overdue?" Currently this requires checking multiple pages — bank reconciliation, pay runs, invoice lists, BAS deadlines. The financial calendar consolidates these into a single, always-accessible timeline.

Additionally, accountants managing multiple clients need a way to inject reminders and tasks for their tenant users (e.g. "Please upload March bank statement", "BAS lodgement due Friday") without phone calls or emails.

# Acceptance Criteria

## AC-1: Date click opens popover
- Clicking the date/time display in the application header opens a popover timeline
- Popover appears below the date, aligned to the header
- Clicking outside or pressing Escape closes the popover
- Popover does not navigate away from the current page

## AC-2: Timeline shows done and upcoming items
- Items are grouped by day with configurable view range (see AC-9)
- Completed items show a tick/checkmark with completion timestamp
- Upcoming items show an open circle with scheduled time
- Overdue items are pinned to the top with red indicator and "X days overdue" label
- Header shows summary badges: green "N done", red "N overdue", amber "N today"
- The "done" count provides positive reinforcement for completed work

## AC-3: Automated financial events
The timeline automatically surfaces these events from existing data:

| Event | Source | Done condition |
|-------|--------|---------------|
| Bank reconciliation | `cr_report` table | `approved = true` for current period |
| Next bank reconciliation | Schedule or last + interval | Shows variance amount if known |
| Payroll / pay run | `pay_run` table + employee pay cycles | `status = 'posted'` |
| Invoice payment reminders | `ar` table, overdue invoices | Payment received |
| Bill due dates | `ap` table, overdue and upcoming bills | Payment made / `force_closed` |
| BAS/GST lodgement | Quarterly schedule (Jul/Oct/Jan/Apr) | BAS report submitted |
| BAS payment | Monthly (21st) or quarterly (same as lodgement) | Payment confirmed |
| Super guarantee | Quarterly (28th of month after quarter end) | SG contributions paid to funds |
| STP finalisation | Annual (14 July) | Finalisation lodged with ATO |
| FBT return | Annual (21 May, or 25 June via tax agent) | Return lodged |
| Period close | End of month/quarter | Period locked |

## AC-4: Accountant-injected tasks
- Users with `master_accountant` or `owner` role can create tasks for the tenant
- Tasks have: title, description (optional), due date, assignee (optional), priority (normal/high)
- Tasks appear in the timeline on their due date
- Any tenant user can mark a task as complete
- Completed tasks show who completed them and when

## AC-5: Collaboration without email
- Accountant creates a task → tenant user sees it in their calendar timeline
- No email notification required (though email notification is a future enhancement)
- Task creator and completer are tracked for audit

## AC-6: Contextual indicators
- The date display in the header shows a subtle badge/dot when there are overdue items
- Badge colour: red for overdue, amber for items due today

## AC-7: Manual event entry by tenant user
- Any authenticated tenant user can create calendar tasks/reminders for themselves
- Tasks have: title, description (optional), due date, priority (normal/high), link (optional)
- Users can edit and cancel their own tasks

## AC-8: Manual event entry by accountant (collaboration)
- Users with `master_accountant` or `owner` role can create tasks visible to all tenant users
- Tasks created by the accountant show "From accountant: {name}" in the description
- This enables lightweight collaboration without phone calls or emails

## AC-9: View range switcher
- The calendar header provides a toggle to switch between three view ranges:
  - **7 Days** — default rolling 7-day window (daily operational view)
  - **Month** — 30-day lookahead (planning view)
  - **Quarter** — 90-day lookahead (compliance and deadline view)
- The selected range is passed to the API as a `?days=` parameter
- Overdue items are always shown regardless of the selected range
- The toggle uses pill-style buttons in the calendar header

## AC-10: Positive reinforcement
- Completed items are counted and displayed as a green "N done" badge in the calendar header
- The done badge appears before the overdue/today badges to lead with positivity
- Completed items remain visible in the timeline (with strikethrough and green checkmark) so users can see their progress

# Future Enhancements

The following are planned for future releases:

- **Full calendar grid view** — month/week grid as a separate page, not popover
- **Recurring task templates** — auto-create tasks on schedules (e.g. monthly bank recon)
- **Email/SMS notifications** — triggered by overdue items or new task assignments
- **Cross-tenant accountant dashboard** — aggregated timeline across all managed tenants
- **Integration with external calendars** — Google Calendar, Outlook sync
- **Audit timeline events** — journal entries, state transitions, reversal chains shown on timeline
- **Asset depreciation run dates** — scheduled depreciation calculations
- **Stocktake dates** — inventory count reminders
- **Insurance/licence renewal reminders** — business compliance dates
- **ATO correspondence deadlines** — response due dates from ATO letters/audits
- **Income tax return lodgement** — varies by agent lodgement program dates

# Out of Scope (Current Release)

- Recurring task templates
- Email/SMS notifications for tasks
- Cross-tenant calendar for accountants
- Integration with external calendars (Google Calendar, Outlook)
- Audit/journal event timeline entries
