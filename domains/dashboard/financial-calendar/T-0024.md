# Task Breakdown

## Phase 1: Database (ledgius-db)

### T-0024.1: Create calendar_task migration

**File**: `migrations/tenant/V1.17__calendar_task.sql`

Create `calendar_task` table with columns: id, title, description, due_date, priority, status, link, created_by, created_by_role, assigned_to, completed_at, completed_by, created_at, updated_at. Add indexes on due_date, status, created_by.

**Verification**: `flyway.sh migrate tenant` succeeds.

## Phase 2: API Domain (ledgius-api)

### T-0024.2: Create calendar domain package

**Directory**: `internal/calendar/`

Files to create:
- `model.go` — `CalendarTask` struct, `TimelineItem` struct, `TimelineResponse` struct
- `repository.go` — CRUD for calendar_task, query helpers for date ranges
- `service.go` — Timeline aggregation: queries cr_report, pay_run, ar, calendar_task; computes BAS/period-close dates; merges and sorts by date
- `handler.go` — HTTP handlers for GET /timeline, POST /tasks, PATCH /tasks/:id, POST /tasks/:id/complete, DELETE /tasks/:id
- `routes.go` — RegisterRoutes binding

### T-0024.3: Wire calendar routes in main.go

Add `calendar.RegisterRoutes(v1, calendarHandler)` in `cmd/server/main.go`.

### T-0024.4: Aggregation queries

Implement these in `service.go`:

1. **Bank recon**: query `cr_report` for latest approved report per bank account, compute next due and variance
2. **Payroll**: query `pay_run` for latest posted run, compute next due from pay cycle
3. **AR reminders**: query `ar` for invoices with `duedate` in window, `force_closed IS NOT TRUE`, group by date
4. **BAS deadlines**: compute from fiscal year config (Q1-Q4 with AU lodgement dates)
5. **Period close**: compute from current month end
6. **Calendar tasks**: query `calendar_task` for open/completed in window

**Verification**: `go test ./internal/calendar/...` passes with table-driven tests for each aggregation.

## Phase 5: Future Enhancements (Backlog)

### T-0024.11: Recurring task templates

Allow users to define recurring tasks (e.g. "Monthly bank reconciliation" repeating on the 5th of each month). Auto-creates calendar_task entries based on RRULE patterns. Requires new `calendar_task_template` table with RRULE field.

### T-0024.12: Email/SMS notifications

Send email notifications when:
- A new task is assigned to a user
- A task becomes overdue
- An accountant creates a task for the tenant

Requires integration with the existing Resend email service.

### T-0024.13: Cross-tenant accountant dashboard

Aggregated calendar view for accountants managing multiple tenants. Shows a unified timeline across all their clients with tenant name labels. Requires platform-level API endpoint that queries multiple tenant DBs.

### T-0024.14: Audit timeline events

Extend timeline to show:
- Journal entry postings
- Entity state transitions (invoice posted, bill approved)
- Reversal/adjustment chains
- Before/after state for audit review

Requires new `audit` category and expanded `TimelineItem` with optional `before`/`after` state fields.

### T-0024.15: Asset depreciation and stocktake

Add timeline events for:
- Scheduled depreciation run dates
- Stocktake/inventory count reminders

Requires asset management domain to be implemented first.

### T-0024.16: External calendar sync

Two-way sync with Google Calendar and Outlook:
- Push Ledgius deadlines to external calendar
- Pull external events into Ledgius timeline

Requires OAuth2 integration with Google/Microsoft.

### T-0024.17: ATO correspondence tracking

Track ATO letters, audit notices, and response deadlines:
- Manual entry of ATO correspondence with due dates
- Auto-reminders as deadlines approach

Requires new `ato_correspondence` table or extension of `calendar_task` with a `source` field.

### T-0024.18: Insurance/licence renewal reminders

Business compliance date tracking:
- Insurance policy renewal dates
- Business licence expiry dates
- Professional registration renewals

Requires new `compliance_date` table for structured recurring business dates.
