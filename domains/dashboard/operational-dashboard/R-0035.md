# Summary

Provide a single-endpoint operational dashboard that returns key financial metrics: AR outstanding, AP outstanding, GST collected, GST paid, net GST position, bank balance, and recent transaction count. The GST position is calculated via the CEL rules engine.

# Problem / Business Context

Business owners and bookkeepers need an at-a-glance view of the business's financial health. The dashboard aggregates key metrics from the general ledger without requiring the user to run individual reports.

# Scope

- Single endpoint returning all dashboard metrics
- AR outstanding (approved, non-return invoices)
- AP outstanding (approved, non-return bills)
- GST collected (from account 2200) and GST paid (from account 1200)
- Net GST position via CEL rules engine
- Bank balance (accounts 1010, 1020, 1030)
- Recent transaction count (last 30 days)

# Out of Scope

- Dashboard widgets/charts (frontend concern)
- Customisable metrics
- Period-specific dashboard (always shows lifetime/current values)
- Cash flow forecast
- Aged receivables/payables summary (available via R-0033 reports)

# Actors / Users

- Business Owner -- daily overview of financial position
- Bookkeeper -- quick status check before entering transactions

# Preconditions

- User is authenticated
- Database has transactions and appropriate accounts configured

# Functional Requirements

- **DASH-001**: System shall return AR outstanding as the sum of approved AR invoice amounts (excluding returns) where amount_bc > 0
- **DASH-002**: System shall return AP outstanding as the sum of approved AP bill amounts (excluding returns) where amount_bc > 0
- **DASH-003**: System shall return GST collected as the negated sum of acc_trans entries against account 2200 (stored as credits, displayed as positive)
- **DASH-004**: System shall return GST paid as the sum of acc_trans entries against account 1200
- **DASH-005**: System shall calculate GST position via CEL rule `dashboard_gst_position` (collected minus paid)
- **DASH-006**: System shall return bank balance as the sum of acc_trans entries against accounts 1010, 1020, 1030
- **DASH-007**: System shall return recent transaction count (approved transactions in the last 30 days)

# Data / Entities / Fields

Output: `Metrics`

| Field | Type | Notes |
|-------|------|-------|
| ar_outstanding | decimal | Sum of approved AR invoices |
| ap_outstanding | decimal | Sum of approved AP bills |
| gst_collected | decimal | GST collected (positive) |
| gst_paid | decimal | GST paid (positive) |
| gst_position | decimal | Net GST (collected - paid) |
| bank_balance | decimal | Sum of bank account balances |
| recent_txn_count | int | Transactions in last 30 days |

# API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | /dashboard | Return all dashboard metrics |

# Validation Rules

None -- endpoint has no input parameters.

# Error Handling

- Database query failures return 500

# Acceptance Criteria

- [x] Single endpoint returns all seven metrics
- [x] AR outstanding excludes returns and unapproved transactions
- [x] AP outstanding excludes returns and unapproved transactions
- [x] GST collected correctly inverted from credit to positive
- [x] GST position calculated via CEL rules engine
- [x] Bank balance from correct account numbers
- [x] Recent transaction count from last 30 days

# Open Questions

1. **Hardcoded account numbers**: Bank accounts (1010, 1020, 1030) and GST accounts (1200, 2200) are hardcoded; should be configurable.
2. **Period filtering**: Dashboard always shows lifetime values; may want period-specific views.
