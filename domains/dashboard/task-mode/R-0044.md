# Summary

Redesigns the Task mode sidebar from a static page-list navigation into a session-based work planner that adapts to accumulated work. The system distinguishes hard deadlines (ATO penalties, employees not paid) from soft cadence work (batch-processable, no guilt) and presents a catch-up summary when the bookkeeper returns after time away.

# Business Context

Many bookkeepers work part-time or batch their work — processing 3-4 days of transactions in a single sitting. The current Task mode sidebar lists pages (Reconcile Bank, Process AR, etc.) as if they are daily tasks, which creates unnecessary pressure when items accumulate.

The key insight: most bookkeeper work is **batch-friendly**. A bookkeeper who reconciles 4 days of bank transactions in one session isn't "behind" — they're working normally. The system should reflect this reality.

# Core Principle: No Guilt UX

The system must never make the user feel guilty for not logging in daily. Accumulated work is presented as "here's what's waiting" not "you're behind". Only genuine hard deadlines (with real consequences) use urgent visual treatment.

# Acceptance Criteria

## AC-1: Hard vs soft deadline distinction

All timeline and task items are classified as one of:

| Type | Visual treatment | Examples |
|------|-----------------|----------|
| **Hard deadline** | Red/urgent, countdown, badge | BAS lodgement, STP finalisation, super guarantee, payroll pay date, FBT return |
| **Soft cadence** | Informational, accumulated volume | Bank reconciliation, receipt recording, invoice chasing, bill review |

Hard deadlines that are missed show as genuinely overdue with consequences explained.
Soft cadence items that accumulate show volume and estimated effort, never "overdue".

## AC-2: Session planner in Task mode

When the user enters Task mode, the sidebar shows a session-based work planner:

**"This Session"** — accumulated work ready to batch-process:
- Shows volume and estimated effort (e.g. "47 transactions, ~20 min")
- Items are clickable, navigating to the relevant page
- Completed items tick off during the session

**"This Week"** — hard deadlines and time-sensitive work:
- Only items with actual due dates within the current week
- Shows countdown (e.g. "Friday", "3 days")

**"Coming Up"** — upcoming deadlines beyond this week:
- Shows date and category
- No urgency treatment

## AC-3: Catch-up summary

When the user opens the app after being away (no activity in 24+ hours), show a brief catch-up banner at the top of the dashboard or task mode:

> "Welcome back — here's what's accumulated since {last_active_date}:"
> - {N} bank transactions to reconcile ({days} days)
> - {N} new bills received
> - {N} invoices now past due (${amount})
> - Payroll due {day} ({countdown})
>
> **Needs attention now:** {hard deadline items only}
> **Ready to batch-process:** {soft cadence items}

## AC-4: Volume and effort estimates

Soft cadence items show accumulated volume with rough effort estimates:
- Bank transactions: count + estimated time (based on ~3 per minute)
- Bills to review: count
- Invoices to chase: count + total amount
- Receipts to record: count

## AC-5: Task mode sidebar restructure

Replace the current page-list sidebar with grouped task categories:

| Group | Items |
|-------|-------|
| **Banking** | Reconcile bank (with volume), Import statement, Review bank rules |
| **Receivables** | Send/chase invoices, Apply credit notes, Write off bad debts, Process refunds |
| **Payables** | Review/approve bills, Schedule payment run, Record supplier credits |
| **Payroll** | Run payroll, Onboard employee, Terminate employee, Lodge STP |
| **Assets** | Buy asset, Sell/dispose asset, Set up loan, Run depreciation |
| **Compliance** | Prepare BAS, Lodge BAS, Super guarantee, FBT return |
| **Period** | Month-end review, Lock period, Year-end close |

Each item shows a badge with accumulated count or upcoming deadline where relevant.

## AC-6: Session completion feedback

When the bookkeeper finishes a batch of work (e.g. reconciles all bank transactions), show a satisfying completion message:

> "Bank reconciled through April 10. 47 transactions matched."

This completion state is reflected in the financial calendar timeline as a completed item.

## AC-7: Carry-forward without guilt

Items not completed in a session simply carry forward to the next session. The system:
- Does NOT mark soft cadence items as "overdue"
- Does NOT send nagging notifications for soft cadence items
- Does show the accumulated volume growing (informational)
- Only escalates hard deadline items as they approach or pass their due date

# Out of Scope

- Gamification (streaks, points, achievements)
- Automated work scheduling or AI-driven prioritisation
- Time tracking per task
- Multi-user task assignment within the session planner
