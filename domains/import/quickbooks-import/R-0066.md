# Summary

Ledgius must be able to import a tenant's accounting data from
QuickBooks Online (QBO), so that a business migrating from QuickBooks
to Ledgius can bring their existing records across. Two delivery
channels are required: **file-based import** (CSV/Excel files
exported from QBO) and **API-based import** (direct pull via the
QuickBooks Online Accounting API using OAuth 2.0). Both channels
feed into the existing Ledgius import staging pipeline (R-0017)
with QuickBooks-specific mapping configurations.

# Terms

| **Term** | Definition. |
|---|---|
| **QBO** | QuickBooks Online — Intuit's cloud-based accounting platform. The Australian edition includes GST, BAS, and STP compliance. |
| **Realm ID** | QuickBooks' unique identifier for a company/organisation, equivalent to a tenant ID. Required in every API request. |
| **IIF** | Intuit Interchange Format — a legacy tab-delimited file format used by QuickBooks Desktop. NOT supported by QuickBooks Online. |
| **Detail Type** | QuickBooks' sub-classification within an account type (e.g., under "Expense" type: "Advertising/Promotional", "Insurance", "Utilities"). Required when importing chart of accounts. |

Vocabulary used in this spec for staging, validation, and commit
(import batch, staging table, commit phase) is defined canonically
in **R-0017 §Terms**.

# Problem / Business Context

QuickBooks Online is a significant competitor in the Australian SMB
accounting market. Businesses migrating to Ledgius from QuickBooks
need a supported path to bring their data across without manual
re-entry. QuickBooks Online Australia includes GST, BAS lodgement,
and STP Phase 2 compliance — so imported data must map correctly
to Ledgius' Australian tax model.

QuickBooks Online does NOT support direct CSV export — reports are
exported as Excel (.xlsx) files which can be converted to CSV. The
import pipeline must handle both Excel and CSV formats. QuickBooks
Desktop uses IIF format which is NOT relevant for QBO migrations.

# Scope

## Supported Import Entities

### v1 — Core data (file-based import)

| Entity | QBO Source | Ledgius Target | Notes |
|---|---|---|---|
| Chart of Accounts | Reports > Account List > Export to Excel | `account` | Columns: Account Name, Type, Detail Type, Account Number. Detail Type maps to Ledgius account category. |
| Customers | Sales > Customers > Export | `entity` + `entity_credit_account` (AR) | Name, email, phone, billing address, ABN, payment terms. |
| Suppliers/Vendors | Expenses > Suppliers > Export | `entity` + `entity_credit_account` (AP) | Name, email, phone, billing address, ABN, payment terms. |
| Invoices | Sales > Invoices > Export | `ar` + `invoice_line` | Invoice number, date, due date, customer, line items, amounts, tax codes, status. Exported as drafts unless marked paid. |
| Bills | Expenses > Bills > Export | `ap` + `invoice_line` | Bill number, date, due date, vendor, line items, amounts, tax codes, status. |
| Tax Codes | Reports > Tax Summary | `tax_code` | AU tax codes: GST, GST-Free (FRE), Input Taxed (INP), BAS Excluded (N-T). Map to Ledgius equivalents. |

### v2 — Extended data (API-based import)

| Entity | QBO API Endpoint | Ledgius Target | Notes |
|---|---|---|---|
| Journal Entries | `GET /v3/company/{realmId}/query?query=SELECT * FROM JournalEntry` | `transactions` + `acc_trans` | Full double-entry journals with memo, date, line items. |
| Payments (received) | `GET /v3/company/{realmId}/query?query=SELECT * FROM Payment` | `acc_trans` | Payment receipts against invoices. |
| Payments (made) | `GET /v3/company/{realmId}/query?query=SELECT * FROM BillPayment` | `acc_trans` | Bill payments against vendor bills. |
| Items/Products | `GET /v3/company/{realmId}/query?query=SELECT * FROM Item` | `parts` | Inventory and service items with SKU, price, tax code. |
| Bank Transactions | `GET /v3/company/{realmId}/query?query=SELECT * FROM Purchase` | `bank_transaction` | Expenses, cheques, credit card charges. |
| Credit Notes | `GET /v3/company/{realmId}/query?query=SELECT * FROM CreditMemo` | `ar` (credit type) | Customer credit notes. |
| Vendor Credits | `GET /v3/company/{realmId}/query?query=SELECT * FROM VendorCredit` | `ap` (credit type) | Supplier/vendor credits. |

## File-Based Import (CSV/Excel)

- **QBI-001**: The import page shall include "QuickBooks Online" as a
  source system option with brand colour (medium green, `#2CA01C`).
- **QBI-002**: The user shall be able to upload Excel (.xlsx) or CSV
  files exported from QuickBooks Online. The system shall auto-detect
  the file format by examining headers and structure.
- **QBI-003**: QBO exports reports as Excel — the system shall parse
  .xlsx files directly without requiring the user to convert to CSV.
- **QBI-004**: Each entity type (accounts, customers, suppliers,
  invoices, bills) shall have a dedicated YAML mapping configuration
  under `internal/importer/mappings/v1/quickbooks/`.
- **QBI-005**: The mapping engine shall handle QBO-specific field
  names and data formats:
  - Date format: `DD/MM/YYYY` (AU locale) or `MM/DD/YYYY` (US locale)
  - Currency: AUD by default for AU tenants
  - Tax codes: `GST`, `GST-Free`, `Input Taxed`, `BAS Excluded`
  - Account types: `Bank`, `Accounts Receivable`, `Other Current Asset`,
    `Fixed Asset`, `Accounts Payable`, `Credit Card`, `Other Current
    Liability`, `Long Term Liability`, `Equity`, `Income`, `Cost of
    Goods Sold`, `Expense`, `Other Income`, `Other Expense`

### Tax Code Mapping

- **QBI-007**: QBO Australian tax codes shall map to Ledgius tax codes:

| QBO Tax Code | Ledgius Tax Code | Rate | BAS Label |
|---|---|---|---|
| GST | GST | 10% | G1 (Total sales), G11 (Capital acquisitions) |
| GST-Free | FRE | 0% | G3 (GST-free sales) |
| Input Taxed | INP | 0% | G4 (Input taxed sales) |
| BAS Excluded | N-T | 0% | Not reported on BAS |
| GST on Imports | IMP | 10% | G15 (Estimated purchases for private use) |

## API-Based Import (OAuth 2.0)

- **QBI-008**: Ledgius shall support connecting to a QuickBooks Online
  organisation via OAuth 2.0 Authorization Code Flow.
- **QBI-009**: The OAuth connection shall use:
  - Authorization URL: `https://appcenter.intuit.com/connect/oauth2`
  - Token URL: `https://oauth.platform.intuit.com/oauth2/v1/tokens/bearer`
  - Scopes: `com.intuit.quickbooks.accounting`
  - Token refresh: access tokens expire in 1 hour, refresh tokens in
    100 days. Proactive refresh at 50 minutes.
- **QBI-010**: Connection credentials (Client ID, Client Secret) shall
  be stored as platform secrets. Per-tenant tokens (access, refresh,
  Realm ID) shall be encrypted at rest using AES-256-GCM (same pattern
  as Xero OAuth in R-0064).
- **QBI-011**: The QBO API uses SQL-like queries for data retrieval.
  The import shall paginate using `STARTPOSITION` and `MAXRESULTS`
  (max 1000 per page).
- **QBI-012**: API rate limits: QBO allows 500 requests per minute per
  Realm ID. The import client shall enforce rate limiting with
  exponential backoff on 429 responses.

### API Import Not Currently Supported

- **QBI-013**: **API-based import is a v2 feature.** The initial
  release supports file-based import only. The OAuth connection
  infrastructure and API client are documented here for future
  implementation but SHALL NOT be built in v1. The UI shall show
  "API import — coming soon" with the connection card greyed out.

## Staging and Validation

- **QBI-014**: All imported data enters the staging pipeline (R-0017)
  before committing to the ledger. No direct writes.
- **QBI-015**: Validation shall check:
  - Required fields present (account name, account type for COA)
  - Tax code mapping exists (unmapped codes flagged as warnings)
  - Duplicate detection (by name, account number, or ABN)
  - Amount formats (strip `$`, commas; handle negative values)
  - Date parsing (handle both AU and US date formats)
- **QBI-016**: The user shall review staged data with a preview grid
  showing source values alongside mapped Ledgius values, with
  warnings highlighted.

## UI / UX

- **QBI-017**: The QuickBooks import option shall appear on the Data
  Import page with:
  - QuickBooks logo/icon
  - Brand colour: medium green (`#2CA01C`)
  - Label: "QuickBooks Online"
  - Subtitle: "Import from CSV/Excel files exported from QuickBooks"
- **QBI-018**: The import workflow follows the existing pipeline:
  1. Select source system (QuickBooks Online)
  2. Upload file(s)
  3. Auto-detect entity type from file structure
  4. Review field mapping (pre-filled from YAML config)
  5. Preview staged data with validation warnings
  6. Commit or discard

# Out of Scope

- QuickBooks Desktop (uses IIF format — a different product entirely)
- QuickBooks Self-Employed (limited API, no full accounting data)
- Real-time sync / bi-directional integration
- Payroll data import (STP data is ATO-bound, not portable)
- Bank feed migration (bank feeds are per-provider, not exportable)

# Actors / Users

- Bookkeeper — performs the import, reviews staged data
- Accountant — may initiate import for client migration
- Business Owner — triggers migration decision

# Preconditions

- Tenant has a Ledgius account with chart of accounts initialised
- User has exported data from QuickBooks Online (for file import)
- For API import (v2): QuickBooks Online OAuth app registered with
  Intuit Developer, Client ID and Secret configured in Ledgius

# Acceptance Criteria

## File Import
- [ ] QuickBooks Online appears as source system on Data Import page (QBI-001)
- [ ] Excel (.xlsx) and CSV files accepted (QBI-002, QBI-003)
- [ ] Chart of accounts imported with correct type mapping (QBI-006)
- [ ] Tax codes mapped to Ledgius equivalents (QBI-007)
- [ ] Customers and suppliers imported with contact details (v1 entities)
- [ ] Invoices and bills imported with line items (v1 entities)
- [ ] All data enters staging pipeline — no direct writes (QBI-014)
- [ ] Validation catches missing required fields, unmapped tax codes (QBI-015)
- [ ] Preview grid shows source vs mapped values (QBI-016)

## API Import (v2 — not in initial release)
- [ ] OAuth 2.0 connection flow works (QBI-008, QBI-009)
- [ ] Token encryption at rest (QBI-010)
- [ ] Paginated data retrieval with rate limiting (QBI-011, QBI-012)
- [ ] API import card shows "coming soon" in v1 (QBI-013)
