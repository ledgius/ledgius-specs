# Summary

Direct API integration with Xero and MYOB AccountRight for pushing
and pulling accounting data without file-based import/export. Users
authenticate via OAuth 2.0, and Ledgius makes API calls on their
behalf to read from or write to the external system. This is an
**additional channel** alongside the existing file-based methods —
it does not replace CSV/AO/CeeData import or CSV/tab-delimited export.

# Problem / Business Context

File-based import/export requires manual steps: export from source
system → download file → upload to Ledgius (or vice versa). This
creates friction, introduces data staleness, and requires users to
navigate two systems. Direct API integration enables:

- **One-click sync** — pull the latest data from Xero/MYOB or push
  Ledgius data directly to the external system
- **Fresher data** — no lag between export and import
- **Reduced error** — no manual file handling, column matching,
  or format conversion
- **Automated scheduling** — future capability for periodic sync

# Out of Scope

- Automated periodic sync / scheduling (future enhancement)
- Real-time webhooks from Xero/MYOB to Ledgius (future enhancement)
- MYOB Essentials or MYOB Business (non-AccountRight) API support
- Xero Projects, Assets, Files, or Payroll APIs
- Two-way conflict resolution (Ledgius is source of truth on push;
  external system is source of truth on pull)

# Actors / Users

- Business Owner — initiates connection, authorises OAuth access
- Bookkeeper — uses push/pull actions, reviews imported data
- Accountant — verifies exported data landed correctly in target system

# Preconditions

- Ledgius tenant has Xero and/or MYOB API credentials configured
  (client_id + client_secret via environment variables or platform admin)
- User has an account on the target system with sufficient permissions
- For MYOB: user must be Admin of the company file (post-March 2025)

# Functional Requirements

## OAuth Connection Management

- **API-001**: System shall support OAuth 2.0 authorization code flow
  for both Xero and MYOB.
- **API-002**: OAuth tokens shall be stored encrypted per tenant ×
  system in the `external_connection` table.
- **API-003**: Access tokens shall be refreshed automatically before
  expiry using the stored refresh token.
- **API-004**: Users shall be able to connect and disconnect external
  systems via the Data mode sidebar.
- **API-005**: Connection status (connected / disconnected / expired)
  shall be visible on the export and import pages.

## API Export (Push)

- **API-006**: Export page shall offer "Push to Xero" and "Push to MYOB"
  as additional channel options alongside file-based export.
- **API-007**: Push shall send entities via the target system's REST API
  using the stored OAuth tokens.
- **API-008**: Push shall use the same mapping engine YAML configs as
  file-based export for field translation.
- **API-009**: Push shall respect rate limits (Xero: 60/min; MYOB:
  rate-limit headers) with automatic backoff.
- **API-010**: Push results shall be recorded in export_run with
  channel='api' for audit trail.

## API Import (Pull)

- **API-011**: Import page shall offer "Pull from Xero" and "Pull from
  MYOB" as additional source options alongside file upload.
- **API-012**: Pull shall read entities from the source system's REST
  API and feed them into the existing import staging pipeline.
- **API-013**: Pulled data shall use the same mapping engine YAML
  configs as file-based import for field translation.
- **API-014**: Pull shall support pagination (Xero uses `page` param;
  MYOB uses `$top` + `$skip`) for large datasets.
- **API-015**: Pull results shall be recorded in import_batch with
  source_system suffixed (e.g. 'xero_api', 'myob_api').

# Security / Permissions

- OAuth tokens stored encrypted at rest (AES-256-GCM via platform key)
- Refresh tokens rotated on each use where the vendor supports it
- Minimum role: Bookkeeper (initiate push/pull)
- Connection management: Business Owner or Bookkeeper
- Disconnect revokes the token at the vendor (Xero/MYOB revocation
  endpoint) and deletes the local record

# Acceptance Criteria

- [ ] OAuth flow works for Xero (connect → callback → tokens stored)
- [ ] OAuth flow works for MYOB (connect → callback → tokens stored)
- [ ] Auto-refresh works before token expiry for both systems
- [ ] Push accounts to Xero via API creates accounts in Xero org
- [ ] Pull accounts from Xero via API populates import staging
- [ ] Push contacts to MYOB via API creates customer/supplier cards
- [ ] Pull contacts from MYOB via API populates import staging
- [ ] Rate limiting respected (no 429 errors on large datasets)
- [ ] Connection status visible on export/import pages
- [ ] Disconnect revokes vendor-side token
