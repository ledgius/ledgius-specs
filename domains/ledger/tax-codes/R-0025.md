# Summary

Manage configurable tax codes that define tax rates, jurisdictions, and linked GL accounts. Tax codes are referenced by invoice and bill line items to calculate GST at the correct rate, replacing the legacy hard-coded 10% assumption.

# Problem / Business Context

Australian businesses must correctly apply GST at various rates (standard 10%, GST-free 0%, input-taxed, export zero-rated). A configurable tax code system allows Ledgius to support multiple tax rates and jurisdictions, and to link each tax code to the appropriate GL account for accurate BAS reporting.

# Scope

- Tax code CRUD (create, read, update)
- Jurisdiction and tax type classification
- Effective date ranges for rate changes
- Linkage to chart of accounts GL account
- Filtering by jurisdiction

# Out of Scope

- Automatic tax rate updates from ATO feeds
- Multi-country tax rule engines
- Tax code deletion (deactivation only)
- Withholding tax codes (PAYG)

# Actors / Users

- Accountant -- configures tax codes and rates
- Bookkeeper -- selects tax codes when creating invoices/bills
- System -- looks up tax rates during invoice/bill calculation

# Preconditions

- User is authenticated
- Chart of accounts is configured with GST Collected and GST Paid accounts

# Functional Requirements

- **TAX-001**: System shall store tax codes with: code (unique), name, description, rate (decimal), jurisdiction, tax_type, linked chart_account_id, effective_from, effective_to, and active flag
- **TAX-002**: Tax code rate shall be stored as a decimal with 4 decimal places precision (numeric(7,4))
- **TAX-003**: System shall support filtering tax codes by jurisdiction
- **TAX-004**: System shall default jurisdiction to AU and tax_type to gst
- **TAX-005**: Tax code creation shall set a default effective_from date of 2000-01-01
- **TAX-006**: System shall support updating name, description, rate, chart_account_id, and active status
- **TAX-007**: Invoice and bill line items shall reference a tax_code_id for rate lookup during GL entry calculation
