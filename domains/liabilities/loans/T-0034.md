# Summary

Implementation plan for the foundation of the loans feature: the `loan_register` schema, backend list + get + non-posting-edit endpoints, and frontend Register + new Detail pages with activity timeline. Foundation task — no payments, accruals, payouts here; those arrive in T-0035/T-0036/T-0037.

# Branch

`feat/loans-register` on `ledgius-specs`, `ledgius-db`, `ledgius-api`, `ledgius-web-app`. Every file created/modified carries `// Spec references: R-0063, A-0044, A-0045, T-0034.`

# Phase 1 — Schema (ledgius-db)

## T-0034.1: loan_register

`V1.YY__loan_register.sql` (next version at commit time):

- Columns: `id uuid pk`, `name text not null`, `description text`, `lender_eca_id int fk entity_credit_account`, `original_amount numeric(20,2) check > 0`, `interest_rate_annual_pct numeric(6,4) check >= 0`, `compounding_frequency text check in ('daily','monthly','quarterly','annually','none')` default `monthly`, `repayment_frequency text check in ('weekly','fortnightly','monthly','quarterly','annually','none')` default `monthly`, `repayment_amount numeric(20,2) check >= 0`, `start_date date not null`, `maturity_date date`, `use_of_funds text check in ('business','mixed','private')` default `business`, `business_use_pct numeric(5,2) check 0..100`, `current_balance numeric(20,2) not null default 0`, `accrued_interest_balance numeric(20,2) not null default 0`, `liability_account_id int fk account`, `accrued_interest_account_id int fk account`, `interest_expense_account_id int fk account`, `status text check in ('draft','active','paid_out','written_off','archived')` default `'active'`, `estimate_changes jsonb not null default '[]'`, `correction_id uuid`, `origination_transaction_id int fk transactions`, `payout_transaction_id int fk transactions`, `payout_date date`, `written_off_transaction_id int fk transactions`, `written_off_date date`, `created_at`, `updated_at`.
- Check: `maturity_date is null or maturity_date > start_date`.
- Check: `payout_date is null or payout_date >= start_date`.
- Check: `written_off_date is null or written_off_date >= start_date`.
- Check: `(use_of_funds = 'mixed') = (business_use_pct is not null)` — mixed-use loans require a business pct.
- Indexes: `(status)`, `(lender_eca_id)`, `(maturity_date)`.
- Comments on every column pointing at the R-0063 LN-### references.

# Phase 2 — Backend (ledgius-api)

## T-0034.3: package skeleton

`internal/loan/`:

- `model.go` — GORM model `Loan`, `Status` + `Method`-like enum types.
- `repository.go` — list with filters + totals, get by id, update-non-posting-fields.
- `service.go` — business commands: `ListLoans`, `GetLoan`, `EditNonPosting`, `ListActivity`. Stubs for `OriginateLoan`, `RecordPayment`, `AccrueInterest`, `PayoutLoan`, `WriteOffLoan` that return `ErrNotImplemented` so the package compiles.
- `handler.go` — HTTP handlers for T-0034 scope.
- `routes.go` — `RegisterRoutes(rg, handler)`.
- `constants.go` — audit entity type = `"loan"` + action constants.
- `*_test.go` — table-driven unit tests per A-0009.

Wire into `cmd/server/main.go` alongside the assets wiring. The `loans` feature gate entry already exists in `DefaultFeatureRouteMap` for `/api/v1/loans` (add if missing).

## T-0034.4: endpoints (this task)

See `openapi-loans.yaml` for shapes.

- `GET  /api/v1/loans` — list with filters (`status`, `lender_id`, `maturity_within_months`, `q` free-text, `include_paid_out`) + header totals (count_active, total_principal_balance, total_monthly_obligation, total_accrued_interest) + pagination.
- `GET  /api/v1/loans/:id` — single loan with lender (joined entity) and derived fields.
- `PATCH /api/v1/loans/:id` — non-posting-field edit (name, description, use_of_funds, business_use_pct). Writes audit `edited`.
- `GET  /api/v1/loans/:id/activity` — audit timeline, newest first.
- `GET  /api/v1/loans/:id/schedule` — **projected** repayment schedule computed from rate + amount + frequency + current balance. No DB write; read-only derivation.

Create / payment / accrual / payout / write-off endpoints are owned by T-0035/T-0036/T-0037 but the service layer stubs land here so the package compiles.

## T-0034.5: tests

- Unit tests on filter combinations and totals SQL — mirror the asset-package pattern.
- Service tests for `EditNonPosting`: audit row written with diff, posting-impacting fields rejected.
- Handler tests via `httptest`: happy path + permission denial + malformed input.
- Schedule derivation tests: straight-line amortisation approximation, interest-only periods.
