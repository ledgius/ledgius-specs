# Summary

Deliver the periodic interest-accrual engine for loans: preview, idempotent run per period, history, reversal. Backend adds endpoints over `interest_accrual_run` / `interest_accrual_run_line` tables. Frontend adds a controls panel on the LoanPaymentsPage (or a dedicated AccrualsPage) for tenants on accrual basis.

Cash-basis tenants (per `tenant_setting.loan_interest_basis='cash'`) skip this engine entirely — interest is recognised at payment time.

Depends on T-0034. Owns the **shared accrual helper** that T-0035 (catch-up before payment) and T-0036 (pro-rata before payout) both call.

# Branch

`feat/loans-interest-accrual`.

# Phase 1 — Schema

## T-0037.1: interest_accrual_run

`V1.YY__interest_accrual_run.sql`:

- Columns: `id uuid pk`, `period_start date`, `period_end date`, `status text check in ('preview','posted','reversed')`, `total_accrued numeric(20,2)`, `parent_transaction_id int fk transactions`, `posted_by uuid`, `posted_at timestamptz`, `reversed_by uuid`, `reversed_at timestamptz`, `reversal_transaction_id int fk transactions`, `created_at`, `updated_at`.
- Unique partial index `(tenant_id, period_end) where status='posted'` — enforces idempotency. (Tenant scoped via schema, so the index becomes `(period_end) where status='posted'`.)

## T-0037.2: interest_accrual_run_line

`V1.YY__interest_accrual_run_line.sql`:

- Columns: `id bigserial pk`, `run_id uuid fk interest_accrual_run`, `loan_id uuid fk loan_register`, `amount numeric(20,2)`, `balance_at_period_start numeric(20,2)`, `days_in_period int`, `interest_rate_used numeric(6,4)`.
- Unique `(run_id, loan_id)`.
- Index `(loan_id)` for per-loan history queries.

# Phase 2 — Calculator helper

## T-0037.3: shared accrual helper

Create `internal/loan/accrual/calculate.go` with pure functions:

```go
type Inputs struct {
    Method        Method // simple | reducing — v1 only supports simple
    Balance       decimal.Decimal
    AnnualRate    decimal.Decimal
    PeriodStart   time.Time
    PeriodEnd     time.Time
}

// Compute returns the simple-interest accrual for [PeriodStart, PeriodEnd].
// PeriodEnd is inclusive; the day count is end-start (so Jan 1..Jan 31 = 30 days).
// Returns zero if rate <= 0 or balance <= 0.
func Compute(in Inputs) decimal.Decimal
```

Plus a `CatchupTo(after, asOf)` convenience that wraps Compute for "catch up from last accrual to a given date" — used by T-0035 (pre-payment) and T-0036 (pre-payout).

Unit tests: zero days, single day, full year, partial period crossing FY end, zero rate, zero balance.

# Phase 3 — Backend engine

### Preview

1. period_start = first-of-month of period_end.
2. Load eligible active loans (skip cash-basis tenants entirely, and per-loan skip if rate=0 OR last_accrual covers this period).
3. For each, compute via the shared calculator.
4. Return `DepreciationPreview`-shaped struct (renamed `AccrualPreview`).

### Run

1. Begin tx.
2. Try INSERT `interest_accrual_run` with status='posted' — unique partial index rejects duplicate per period.
3. Recompute preview inside tx.
4. INSERT lines.
5. Call `journal.Post(ctx, journal.LoanAccrualRunCmd{run, lines})` per A-0045 §3.
6. Update each loan's `accrued_interest_balance += line.amount`.
7. `audit.Write(entity='accrual_run', action='posted', payload={period, total, loan_count})`.
8. COMMIT.

### Reverse

1. Validate `status='posted'`.
2. Begin tx.
3. Call `journal.Post(ctx, journal.LoanAccrualReversalCmd{source_run_id})` — exact inverses.
4. Update each affected loan's `accrued_interest_balance -= line.amount`.
5. UPDATE run: `status='reversed'`, `reversed_by`, `reversed_at`, `reversal_transaction_id`.
6. Audit row.
7. COMMIT.

### List

Paginated list with filters. Per-run detail endpoint includes lines.

## T-0037.5: endpoints

- `GET  /api/v1/loans/accrual/preview?period_end=YYYY-MM-DD` (`loans:view`).
- `POST /api/v1/loans/accrual/run` (`loans:run_accrual`) — body `{period_end}`.
- `POST /api/v1/loans/accrual/runs/:run_id/reverse` (`loans:reverse_accrual`) — body `{reason}`.
- `GET  /api/v1/loans/accrual/runs` — list.
- `GET  /api/v1/loans/accrual/runs/:run_id` — detail with lines.
- `GET  /api/v1/loans/:id/accrual-history` — per-loan history.

## T-0037.6: tests

- Calculator: see §T-0037.3.
- Service: preview returns correct rows, mixed methods + zero rates handled. Run idempotent (second-run rejected). Run with zero eligible loans posts no journal but writes the run row. Reverse run flips status + restores balances.
- Cash-basis tenants: preview returns empty; run rejected with `ErrCashBasisTenant`.
- Period-lock rejection.
- Journal correctness per A-0045 §3 + §7.

## T-0037.7: nightly reconciliation job

Compare `loan_register.accrued_interest_balance` with `sum(acc_trans amount)` tagged `loan_id` on the loan's accrued-interest-payable account (minus reversals). Emits a metric on mismatch. Logs structured fields per A-0045 §12. Reports only — does not auto-correct.

## T-0037.9: accruals controls

- Past runs list with View detail + Reverse actions.
