# Summary

Native iOS and Android mobile apps for Ledgius providing field/on-the-go functionality: create quotes, create and send invoices, accept payments, capture receipt photos, and manage contacts. The apps connect to the existing Ledgius API (`api.ledgius.com`) and support Stripe payment processing — either via Ledgius's master Stripe account (Stripe Connect) or the user's own Stripe account.

# Business Context

Tradespeople, sole traders, and field-based small business owners need to:
- Quote a job on-site and send it to the customer immediately
- Invoice the customer as soon as the job is done
- Accept payment on the spot (tap-to-pay, payment link, or QR code)
- Photograph receipts for expenses before they lose them
- Add new customers/suppliers while in the field

These are high-frequency, time-sensitive actions that can't wait until the bookkeeper is at a desk. The mobile app is NOT a full accounting app — it's a field companion for the web app.

# Core Principle: Field-First, Sync Later

The mobile app prioritises speed and reliability in the field:
- **Offline-capable**: quotes, invoices, and receipt captures work without internet
- **Sync when connected**: data syncs to the API when connectivity is available
- **Minimal input**: large buttons, smart defaults, auto-fill from recent data
- **Camera-first**: receipt capture is one tap from the home screen

# Acceptance Criteria

## AC-1: Create Quotes

Create and send quotes from the mobile device:

- **Quick create**: customer (search/select or create new), line items (description + quantity + price), notes
- **Templates**: reuse previous quotes as templates
- **Tax**: GST auto-calculated per line item using tax codes from the API
- **Totals**: subtotal, GST, total displayed in real-time
- **Send**: email quote to customer as branded PDF (via API → Resend)
- **Status**: draft → sent → accepted → converted to invoice
- **Convert**: one-tap convert accepted quote to invoice
- **Offline**: create and save locally, send when connected

## AC-2: Create Invoices

Create, send, and track invoices:

- **Quick create**: same flow as quotes but creates an invoice (AR transaction)
- **From quote**: convert an accepted quote to an invoice with one tap
- **Due date**: auto-calculated from customer payment terms
- **Invoice number**: auto-generated, sequential per tenant
- **Send**: email invoice to customer as branded PDF with payment link
- **Payment link**: embedded Stripe payment link in the email and on the invoice PDF
- **Status tracking**: draft → sent → viewed → paid → overdue
- **Push notifications**: notify when customer views or pays the invoice
- **Offline**: create and save locally, send when connected

## AC-3: Provide Receipt for Payments

Generate and send payment receipts:

- **Auto-generated**: when a payment is recorded (manually or via Stripe), a receipt is created automatically
- **PDF receipt**: branded receipt with payment details, GST breakdown, business ABN
- **Send**: email or SMS receipt to customer
- **Share**: native share sheet (AirDrop, Messages, WhatsApp, email)
- **Print**: support AirPrint / Bluetooth receipt printers for tradies
- **Receipt number**: auto-generated, sequential

## AC-4: Capture Receipts

Photograph expense receipts for later matching:

- **Camera capture**: one-tap camera access from home screen
- **Smart crop**: auto-detect receipt edges, crop and straighten
- **OCR extraction**: extract merchant name, date, amount, GST from the photo (on-device or via API)
- **Quick tag**: assign to an expense category (or leave untagged for later)
- **Bulk capture**: capture multiple receipts in sequence without returning to home
- **Sync**: upload photos to API with extracted data
- **Web app matching**: receipts appear in the web app's expense documentation section for matching to bank transactions
- **Storage**: photos stored in cloud (S3/R2) linked to the tenant

## AC-5: Create Contact

Add customers and suppliers from the field:

- **Quick create**: name, email, phone, ABN (optional)
- **Type**: customer or supplier (default: customer)
- **From phone contacts**: import from device contact book
- **Business card scan**: OCR from a business card photo (future enhancement)
- **Sync**: immediately syncs to API, available in web app
- **Recent contacts**: show recent contacts at top of selection lists

## AC-6: Direct Payment via Stripe

Accept payments on mobile devices:

### Option A: Ledgius Stripe Connect (default)
- Ledgius operates as a Stripe Connect platform
- User's customers pay via Stripe — funds flow through Ledgius's Stripe account then settle to the user's bank
- Ledgius takes a small platform fee (e.g. 0.5% on top of Stripe's standard fees)
- User doesn't need their own Stripe account — zero setup friction
- Settlement: daily or weekly to user's linked bank account

### Option B: User's Own Stripe Account
- User connects their existing Stripe account via OAuth (Stripe Connect Standard)
- Payments go directly to their Stripe account
- No Ledgius platform fee
- User manages their own Stripe dashboard for disputes, refunds, etc.

### Payment methods:
- **Payment link**: generated per invoice, shareable via SMS/email/QR code
- **QR code**: customer scans, pays on their phone
- **Tap-to-pay**: iPhone Tap to Pay / Android NFC for in-person card payments (future — requires Stripe Terminal SDK)
- **Manual entry**: record a cash/bank transfer payment manually

### Payment recording:
- When Stripe processes a payment: webhook updates invoice status → auto-generates receipt → pushes notification to mobile app
- Payment creates the corresponding journal entry in the ledger (receipt → debit bank, credit AR)

# Tech Stack Considerations

## Cross-platform approach options:

| Approach | Pros | Cons |
|----------|------|------|
| **React Native** | Share code with web app (React), large ecosystem, Expo for fast dev | Performance for camera/NFC, native feel |
| **Flutter** | Excellent performance, single codebase, great camera/NFC support | Different language (Dart), no code sharing with web |
| **Native (Swift + Kotlin)** | Best performance, best UX, full platform access | Two codebases to maintain, slower development |

**Recommendation**: React Native with Expo — maximises code and knowledge sharing with the existing React web app. Camera and Stripe SDK have mature React Native packages.

## API integration:
- Same API: `api.ledgius.com`
- Same auth: JWT tokens (same login, Google SSO works on mobile)
- Same endpoints: `/invoices`, `/contacts`, `/receipts`, etc.
- New endpoints needed: `/quotes`, `/receipt-captures` (photo upload), `/payments/stripe-connect`

## Offline support:
- Local SQLite database for offline-first data
- Sync queue: pending creates/updates queued and sent when online
- Conflict resolution: server wins for most fields, merge for line items

# Out of Scope (V1)

- Full accounting operations (journals, reconciliation, BAS — desktop only)
- Inventory management
- Payroll
- Bank feeds connection management
- Multi-currency on mobile
- Apple Pay / Google Pay (as payment acceptance — only Stripe for V1)
- Tap-to-pay NFC (requires Stripe Terminal integration — V2)
- Business card OCR scanning
- Expense claim workflows
- Team management / multi-user on mobile
