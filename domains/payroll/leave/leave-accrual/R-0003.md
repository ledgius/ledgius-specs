# Summary

Track employee leave entitlements, accruals, and usage. Calculate leave balances per pay cycle and deduct when leave is taken. Support annual leave, personal/carer's leave, and long service leave per the National Employment Standards.

# Problem / Business Context

Australian employers are legally required to provide minimum leave entitlements under the National Employment Standards (NES) in the Fair Work Act 2009. Full-time employees accrue 4 weeks annual leave and 10 days personal/carer's leave per year. Long service leave is governed by state and territory legislation. Ledgius must accurately accrue, track, and report leave balances to comply with these obligations, support STP Phase 2 reporting, and calculate correct termination payouts.

# Scope

- Leave accrual calculation per pay cycle for annual, personal/carer's, and long service leave
- Leave balance tracking per employee per leave type
- Leave deduction when leave is taken in a pay run
- Leave payout calculation on termination
- Leave loading (17.5%) on annual leave payout
- Leave balance reporting for STP Phase 2
- Configurable negative balance policy (allow/disallow leave in advance)

# Out of Scope

- Leave application/approval workflows (request, manager approval, calendar)
- Leave calendar or rostering integration
- Parental leave, compassionate leave, community service leave (future NES extensions)
- Public holiday management
- Leave accrual for contractors (contractors have no leave entitlements)

# Actors / Users

- Business Owner -- manages leave policies and reviews balances
- Bookkeeper -- records leave taken in pay runs, reviews accruals
- Accountant -- verifies leave liability for financial reporting
- Employee (future) -- views own leave balance (self-service, not in initial scope)

# Preconditions

- Employee records exist with employment type (full_time, part_time, casual)
- Pay run processing is operational
- Leave type configuration is established

# Functional Requirements

- **PAY-LEA-001**: System shall accrue annual leave per pay cycle based on employment type
  - Full-time: 4 weeks/year = 152 hours/year, accrued proportionally each pay cycle
  - Part-time: pro-rata based on ordinary hours
  - Casual: nil (casual loading compensates)
- **PAY-LEA-002**: System shall accrue personal/carer's leave per pay cycle
  - Full-time: 10 days/year = 76 hours/year
  - Part-time: pro-rata based on ordinary hours
  - Casual: nil
- **PAY-LEA-003**: System shall track leave balances per employee per leave type (annual, personal, long_service)
- **PAY-LEA-004**: System shall deduct leave balances when leave is recorded in a pay run
- **PAY-LEA-005**: System shall prevent leave exceeding available balance by default, with a configurable option to allow negative balances for specified leave types
- **PAY-LEA-006**: System shall support long service leave accrual at state-dependent rates (typically 8.67 weeks after 10 years continuous service)
- **PAY-LEA-007**: System shall support leave loading (17.5%) on annual leave payout where applicable per award
- **PAY-LEA-008**: System shall support leave in advance (negative balance) up to a configurable limit per leave type
- **PAY-LEA-009**: System shall calculate leave payout on termination -- accrued but untaken annual leave must be paid out; personal/carer's leave is not paid out
- **PAY-LEA-010**: System shall report leave balances (annual, personal, long service) in STP submissions per employee

# Data / Entities / Fields

Primary entity: `LeaveBalance`

| Field | Type | Notes |
|-------|------|-------|
| id | int (PK) | Auto-increment |
| employee_id | int (FK) | References Employee |
| leave_type | string | annual, personal, long_service |
| accrued_hours | decimal | Total hours accrued to date |
| taken_hours | decimal | Total hours taken to date |
| balance_hours | decimal | Computed: accrued_hours - taken_hours |
| as_at_date | date | Balance effective date |

Related entities: `Employee` (employment type drives accrual rates), `PayRunLine` (leave_accrued, leave_taken JSONB fields)

# UX / UI Behaviour

- Leave balance section displayed on the employee detail page showing current balances per leave type
- Leave balances formatted in hours with decimal precision (e.g. "76.00 hrs")
- Negative balances displayed in red text
- Leave taken recorded via pay run line items, not a separate leave request screen
- Leave payout calculated and displayed during termination workflow

# Validation Rules

- Leave type must be one of: annual, personal, long_service
- Accrued hours and taken hours must be non-negative
- Leave deduction must not exceed balance unless negative balance is explicitly permitted for the leave type
- Leave payout on termination only applies to annual leave (not personal)

# Error Handling

- Attempt to deduct leave exceeding balance (when negative not allowed) returns validation error with current balance
- Invalid leave type returns error with allowed values
- Leave accrual for casual employees returns zero (not an error)

# Security / Permissions

- Minimum role: Bookkeeper (view and manage leave balances)
- Leave balance changes audit-logged
- Leave balances visible on employee detail page per existing employee access controls

# Acceptance Criteria

- [ ] Annual leave accrues correctly per pay cycle for full-time employees (152 hrs/year proportional)
- [ ] Annual leave accrues pro-rata for part-time employees
- [ ] Casual employees accrue zero leave
- [ ] Personal/carer's leave accrues correctly per pay cycle (76 hrs/year for full-time)
- [ ] Leave balance displayed on employee detail page
- [ ] Leave deducted when recorded in a pay run
- [ ] Negative balance prevented unless explicitly configured
- [ ] Termination calculates annual leave payout (personal leave excluded)
- [ ] Leave loading (17.5%) applied on annual leave payout where configured
- [ ] Leave balances reported in STP submissions

# Regulatory References

- Fair Work Act 2009 -- National Employment Standards (Part 2-2, Divisions 6-7)
- NES Annual Leave: 4 weeks (full-time), pro-rata (part-time)
- NES Personal/Carer's Leave: 10 days/year (full-time), pro-rata (part-time)
- Long Service Leave: State and territory legislation (varies by jurisdiction)
- STP Phase 2: Leave balances must be reported per employee per submission

# Open Questions

1. **State-based LSL rules**: Long service leave rates and qualifying periods vary by state. Need to determine how to model state jurisdiction per employee.
2. **Award-specific loading**: Leave loading of 17.5% applies under some awards but not all. Need configurable per-award or per-employee setting.
3. **Leave balance rollover**: Annual leave accumulates indefinitely under NES, but some awards allow employers to direct employees to take excessive leave. Not yet modelled.
