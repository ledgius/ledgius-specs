# Summary

Process employee payroll for a given period -- calculate gross pay, PAYG withholding, superannuation guarantee, leave accruals, and net pay. Generate GL journal entries and prepare data for STP Phase 2 reporting. The pay run is the central orchestration point for all payroll calculations.

# Problem / Business Context

Australian employers must pay employees correctly and on time per the Fair Work Act, withhold PAYG tax per ATO Schedule 1, contribute superannuation per the SG Act, and report all payroll events via STP. The pay run engine must coordinate these obligations into a single atomic process that produces correct, auditable, and immutable payroll records with corresponding general ledger entries.

# Scope

- Pay run creation for a specified period (start date, end date, payment date)
- Employee selection (default: all active employees matching pay cycle)
- Gross pay calculation from employee pay rates and hours/units
- PAYG withholding calculation using ATO tax table coefficients
- Superannuation guarantee calculation on OTE
- Net pay calculation (gross - PAYG - deductions)
- GL journal entry creation for each pay run
- Pay run lifecycle management (draft, finalised, submitted)
- Pay run immutability after finalisation
- Pay run list and detail pages
- Idempotent and deterministic calculation

# Out of Scope

- Payslip generation (PDF)
- Bank file generation (ABA format) for pay disbursement
- Award rate lookup engine (rates are manual per employee)
- Rostering / timesheet integration
- Multi-currency payroll
- Contractor invoicing (contractors excluded from pay runs)

# Actors / Users

- Business Owner -- initiates and reviews pay runs
- Bookkeeper -- creates draft pay runs, reviews calculations, finalises
- Accountant -- verifies GL journals posted by pay runs, reviews compliance

# Preconditions

- Employee records exist with pay rates, tax profiles, and super details
- PAYG tax brackets loaded for the relevant financial year
- Super guarantee rates loaded for the relevant financial year
- Chart of Accounts configured with payroll expense, liability, and bank accounts
- GL journal posting is operational

# Functional Requirements

- **PAY-RUN-001**: System shall create a pay run for a specified period (start date, end date, payment date)
- **PAY-RUN-002**: System shall allow selection of employees to include in a pay run; default is all active employees matching the pay cycle
- **PAY-RUN-003**: System shall calculate gross pay from employee pay rates and hours/units
- **PAY-RUN-004**: System shall calculate PAYG withholding using ATO tax table coefficients per the formula: `withholding = (a * weekly_earnings) - b`
- **PAY-RUN-005**: System shall calculate superannuation guarantee on Ordinary Time Earnings (OTE) using the rate from the SuperGuaranteeRate table; OTE excludes overtime and bonuses (configurable)
- **PAY-RUN-006**: System shall calculate net pay as gross pay minus PAYG withholding minus other deductions
- **PAY-RUN-007**: System shall create GL journal entries for each pay run -- debits to wage expense accounts, credits to PAYG payable, super payable, and bank
- **PAY-RUN-008**: Pay run results shall be immutable once finalised; corrections must be made via adjustment pay runs, not edits
- **PAY-RUN-009**: System shall support the pay run lifecycle: draft (editable) -> finalised (locked, GL posted) -> submitted (STP sent)
- **PAY-RUN-010**: System shall support pay run reversal by creating a reversing journal entry
- **PAY-RUN-011**: System shall calculate leave accruals per pay cycle during pay run processing
- **PAY-RUN-012**: System shall support adjustments and backpay in a subsequent pay run
- **PAY-RUN-013**: Pay run list shall display period dates, payment date, status, totals (gross, tax, super, net), and employee count
- **PAY-RUN-014**: Pay run detail shall show per-employee breakdown of gross, PAYG, super, net, hours, and rate
- **PAY-RUN-015**: Pay run calculation shall be idempotent -- re-running with the same inputs produces the same result
- **PAY-RUN-016**: Pay run for 50 employees shall complete in under 5 seconds (non-functional)
- **PAY-RUN-017**: Pay run calculation shall be deterministic -- same inputs always produce same outputs (non-functional)

# UX / UI Behaviour

- Pay run list page shows period, payment date, status, totals, and employee count
- Clickable rows navigate to pay run detail page
- Pay run detail page shows per-employee breakdown with gross, PAYG, super, net, hours, and rate
- Draft pay runs show "Finalise" button in header; finalised pay runs are read-only
- Finalised pay runs show "Submit to STP" action
- Status stepper: Draft -> Finalised -> Submitted

# Validation Rules

- Pay period start must be before pay period end
- Payment date must be on or after pay period end
- At least one employee must be included in a pay run
- Employee must have at least one pay rate defined
- Gross pay, PAYG, super, and net must all be non-negative
- Status transitions must follow lifecycle: draft -> finalised -> submitted (no skipping or reversal)

# Error Handling

- Missing pay rate for an employee returns validation error identifying the employee
- Missing tax bracket for the calculation period returns error with the date and residency status
- Missing super rate for the calculation period returns error with the date
- Attempt to edit a finalised pay run returns immutability error directing user to create an adjustment run

# Security / Permissions

- Minimum role: Bookkeeper (create, view, finalise pay runs)
- Pay run creation and finalisation audit-logged
- GL journal entries created by pay runs are linked to the pay run for traceability

# Acceptance Criteria

- [x] Pay run creates with period dates and selected employees
- [x] PAYG calculated from tax tables using coefficient formula
- [x] Super calculated from SG rate on OTE
- [x] GL journal entries created on finalisation
- [ ] Pay run list shows period, payment date, status, totals, employee count
- [ ] Pay run detail shows per-employee breakdown
- [ ] Finalised pay runs are immutable
- [ ] Draft pay runs can be re-calculated (idempotent)
- [ ] Pay run lifecycle enforced (draft -> finalised -> submitted)
- [ ] Leave accruals calculated during pay run
- [ ] Pay run reversal creates reversing journal

# Regulatory References

- ATO NAT 1004 Schedule 1: PAYG withholding formula and coefficients
- Superannuation Guarantee (Administration) Act 1992: SG on OTE
- Fair Work Act 2009: Minimum pay rates per applicable award
- STP Phase 2: Every finalised pay run must generate an STP submission event

# Open Questions

1. **Pay component granularity**: Currently calculates a flat gross. Needs to support salary, overtime, allowances, bonuses as separate components for STP Phase 2 reporting.
2. **Award rate lookup**: No award engine yet -- rates are manual per employee. May need a rules engine for rate determination.
3. **EOFY boundary**: Pay runs spanning end of financial year need special handling for tax table transitions.
