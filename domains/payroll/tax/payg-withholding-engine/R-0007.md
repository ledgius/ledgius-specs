# Summary

Calculate PAYG (Pay As You Go) withholding amounts for employee pay using ATO Schedule 1 tax table coefficients. The engine stores tax brackets per financial year, handles resident and non-resident tax tables, tax-free threshold claims, pay cycle conversions, no-TFN withholding, and Stage 3 tax cut changes from FY2024-25.

# Problem / Business Context

Australian employers are legally required to withhold income tax from employee payments under the PAYG withholding system administered by the ATO. The withholding amount is calculated using the ATO's Schedule 1 (NAT 1004) coefficient formula, which varies by residency status, tax-free threshold claim, and financial year. Tax tables change each financial year (effective 1 July), and the Stage 3 tax cuts from FY2024-25 significantly altered bracket thresholds and rates. Incorrect withholding results in ATO penalties for the employer and incorrect tax outcomes for employees.

# Scope

- PAYG tax bracket storage per financial year with coefficients a and b
- Withholding calculation using formula: `withholding = (a * weekly_earnings) - b`
- Resident and non-resident tax tables
- Tax-free threshold claimed and not claimed variants
- Pay cycle conversion (fortnightly, monthly to weekly equivalent)
- No-TFN withholding rate (47%)
- Stage 3 tax cut bracket changes from FY2024-25
- Historical tax tables FY2020-21 through FY2025-26
- PAYG configuration display in frontend with FY selector

# Out of Scope

- HELP/HECS repayment calculation (separate repayment thresholds table needed)
- SFSS (Student Financial Supplement Scheme) repayment calculation
- Medicare levy surcharge calculation
- Tax variation amounts (employee-requested reduced withholding)
- Tax offset calculations
- Foreign resident withholding for non-employment income
- Working Holiday Maker specific tax rates (WHM rates differ from standard non-resident)
- Editing tax brackets via UI (currently seed data only)

# Actors / Users

- Business Owner -- relies on correct tax withholding for compliance
- Bookkeeper -- reviews PAYG calculations in pay runs, configures employee tax profiles
- Accountant -- verifies PAYG withholding accuracy, reviews tax table configuration
- ATO -- defines Schedule 1 coefficients; receives PAYG via STP

# Preconditions

- PAYG tax brackets loaded for the relevant financial year(s)
- Employee records include residency status, tax-free threshold claim, and TFN status
- Pay run processing is operational

# Functional Requirements

- **PAY-TAX-001**: System shall store PAYG tax brackets per financial year with coefficients a and b; formula: `withholding = (a * weekly_earnings) - b`
- **PAY-TAX-002**: System shall support separate tax tables for resident and non-resident employees
- **PAY-TAX-003**: System shall support tax-free threshold claimed and not claimed variants
- **PAY-TAX-004**: System shall select the correct bracket based on weekly earnings amount, residency status, tax-free threshold claim, and effective date
- **PAY-TAX-005**: System shall convert fortnightly and monthly pay to weekly equivalent for calculation (fortnightly / 2, monthly * 12 / 52)
- **PAY-TAX-006**: System shall store tax tables for FY 2020-21 through FY 2025-26 (68 rows, 11-12 brackets per FY)
- **PAY-TAX-007**: System shall correctly reflect Stage 3 tax cut changes from FY 2024-25 (16c replaces 19c bracket, 30c replaces 32.5c, 37c threshold raised)
- **PAY-TAX-008**: System shall apply no-TFN withholding rate of 47% when employee has not provided a TFN
- **PAY-TAX-009**: System shall support HELP debt repayment calculation using separate HELP repayment threshold tables
- **PAY-TAX-010**: System shall support tax variation amounts where an employee has ATO approval for reduced withholding

# Data / Entities / Fields

Primary entity: `PAYGTaxBracket`

| Field | Type | Notes |
|-------|------|-------|
| id | int (PK) | Auto-increment |
| bracket_name | string | Descriptive name for the bracket |
| residency | string | resident, non_resident |
| tax_free_claimed | bool | Whether tax-free threshold is claimed |
| weekly_from | decimal | Lower bound of weekly earnings (inclusive) |
| weekly_to | decimal (nullable) | Upper bound (NULL = no upper limit) |
| coefficient_a | decimal(10,4) | Multiplier in withholding formula |
| coefficient_b | decimal(10,4) | Subtractor in withholding formula |
| effective_from | date | Start of financial year |
| effective_to | date (nullable) | End of financial year |

Seed data: 68 rows covering FY 2020-21 through FY 2025-26

# UX / UI Behaviour

- PAYG configuration page displays tax brackets grouped by financial year
- FY selector chips allow switching between financial years
- Each bracket row shows: bracket name, weekly range, coefficients, and annualised equivalent
- Weekly and annual display modes available
- Read-only display (brackets are seed data, not user-editable)

# Validation Rules

- Residency must be one of: resident, non_resident
- Weekly earnings must be non-negative
- Coefficient a must be non-negative
- Effective dates must not overlap for the same residency/threshold combination
- Weekly range bounds must not overlap within the same FY/residency/threshold combination

# Error Handling

- No matching bracket for given earnings/residency/date returns error with lookup parameters for debugging
- Employee without TFN triggers 47% flat rate withholding (not an error, but logged)
- Missing tax tables for the financial year returns error directing admin to load tables

# Security / Permissions

- PAYG calculations audit-logged as part of pay run audit trail

# Regulatory References

- ATO NAT 1004: Schedule 1 -- Statement of formulas for calculating amounts to be withheld
- Income Tax Rates Act 1986: Marginal tax rates and thresholds
- Treasury Laws Amendment (Stage 3 Tax Cuts): Effective 1 July 2024, 16c replaces 19c bracket, 30c replaces 32.5c bracket, 37c threshold raised to $135,000
- HELP/HECS: Higher Education Loan Program repayment thresholds (separate from PAYG, not yet implemented)
- No-TFN withholding: 47% flat rate per ATO guidelines

# Open Questions

1. **HELP repayment thresholds**: Need to implement a separate HELP repayment thresholds table with annual updates. HELP repayment is calculated on total earnings, not per-pay, which adds complexity.
2. **Medicare levy surcharge**: MLS applies to higher earners without private health insurance. Requires employee declaration of PHI status -- not yet modelled.
3. **Tax table updates**: Currently seed data only. Need a process for loading new FY tax tables (either admin UI or migration script each July).
