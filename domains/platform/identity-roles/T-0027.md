# Terms

Vocabulary used in this task plan (population, role, active role, scope,
app surface, etc.) is defined canonically in **R-0052 §Terms**.

# Summary

Phased implementation plan for R-0052 (four-population identity model + universal
audit fields) and R-0053 (platform-owner pricing plan administration). The
two are sequenced because R-0053 depends on the `platform_owner` population
defined by R-0052.

Every source file modified or created under this task carries the
`// Spec references: R-0052, A-0028.` (or R-0053, A-0029) provenance header
per the project Provenance rule.

# Branch

`feat/identity-roles-and-pricing-plans` on each impacted repo.

# Phase 2 — Backend identity primitives (R-0052)

## T-0027.5: `internal/identity/` package
- `model.go` — Population, PopulationAssignment, Region
- `repository.go` — assignment lookup, region tree walk
- `service.go` — `EnsureCustomerAssignment`, `GrantAccountantAccess`,
  `RevokeAssignment`, `ResolveAvailablePopulations(userID)`

## T-0027.6: JWT claim shape extension
- `pkg/auth/token.go` updated to include `active_role`, `active_role_scope_id`,
  `available_roles[]`
- `POST /api/v1/auth/login` returns the new shape
- `POST /api/v1/auth/switch-role` mints a new token after validating the
  requested population assignment is held + un-revoked
- Verification: integration tests against a sandbox tenant set

## T-0027.7: `middleware.LoadActiveRole` + `LoadAppSurface` + `AuditContext`
- New middleware chain steps per A-0028 §"Middleware pipeline"
- `X-Client-App` header parsing with `api` fallback
- Mobile-only-customer enforcement (IDR-008/010)
- Verification: unit tests on each middleware in isolation

## T-0027.8: `pkg/audit` updates
- `audit.WriteFromCtx(ctx, action, entity, before, after)` populates the
  five universal fields automatically
- All existing audit call sites migrated to the new helper (mechanical
  rewrite per package)
- Verification: code search for old audit signatures returns zero hits

# Phase 3 — Backfill + R-0041 supersession

## T-0027.9: Backfill script — population assignments
- For every row in `tenant_memberships`: insert `population_assignments`
  with `population='customer'`, `tenant_id`, `intra_role` from the
  membership row
- Idempotent (`ON CONFLICT DO NOTHING`)
- Verification: count of pre-migration tenant_memberships ==
  customer-population assignments

## T-0027.10: Backfill script — historical audit_log
- `UPDATE audit_log SET active_role='customer', app_surface='web',
  actor_label='backfill-r0052' WHERE active_role IS NULL`
- Run in batches of 10k with sleep to avoid blocking
- Verification: zero rows with NULL `active_role` after backfill

## T-0027.11: Mark R-0041 as Superseded
- Update R-0041 frontmatter: `Status: Superseded`, `Superseded By: R-0052`
- No code change

# Phase 4 — Frontend identity (R-0052)

## T-0027.12: PopulationPicker + role-switch UI
- New component shown post-login when `available_populations.length > 1`
