# Summary

Architecture for the platform administration console. Operates
against the `ledgius_platform` database for tenant/user/plan
management, and dynamically creates/operates on tenant databases
for provisioning and backup operations.

# Requirement Link

- R-0068 (Platform Administration Console)

# Technical Context

Ledgius uses database-per-tenant isolation. Each tenant has:
- A record in `ledgius_platform.tenant`
- An isolated PostgreSQL database (named by slug)
- A filesystem directory (`/data/{slug}/`)

The platform admin console needs to operate across all three layers:
platform DB reads/writes, tenant DB creation/backup/restore, and
filesystem operations.

## Tenant Provisioning Pipeline

```
Admin clicks "Accept" on sign-up
    ↓
1. Create/complete tenant record in platform DB
   - Generate slug from business name
   - Set status = 'active' (or 'trial' if trial period)
   - Set is_test flag
   - Assign plan
    ↓
2. Create PostgreSQL database
   - CREATE DATABASE {slug}
   - Run Flyway migrations (V1.00 → latest)
   - Seed default data (COA template, tax codes, defaults)
   - Expected: 20-30 seconds
    ↓
3. Create filesystem directory
   - mkdir /data/{slug}/
   - Set permissions
    ↓
4. Create tenant users
   - Insert owner user in platform_user
   - Insert owner user in tenant DB user table
   - Send welcome email
    ↓
5. Return: tenant ready
```

### Slug Algorithm

```go
func GenerateSlug(businessName string) string {
    s := strings.ToLower(businessName)
    s = regexp.MustCompile(`[^a-z0-9\s-]`).ReplaceAllString(s, "")
    s = regexp.MustCompile(`\s+`).ReplaceAllString(s, "-")
    s = strings.Trim(s, "-")
    // Ensure uniqueness — append -2, -3, etc. if slug exists
    return s
}
```

## Feature Gate Architecture

### API Layer (Middleware)

```go
// FeatureGate middleware checks if the tenant's plan includes
// the required feature before allowing the request.
func RequireFeature(feature string) gin.HandlerFunc {
    return func(c *gin.Context) {
        tenantID := middleware.TenantIDFromContext(c)
        if !featureStore.IsEnabled(tenantID, feature) {
            response.Forbidden(c, "Feature not available on your plan")
            c.Abort()
            return
        }
        c.Next()
    }
}
```

### UI Layer (React)

```tsx
function useFeatureGate(feature: string): boolean {
    const { data: features } = useQuery({
        queryKey: ["features"],
        queryFn: () => api.get<string[]>("/features"),
    })
    return features?.includes(feature) ?? false
}

// Usage in components:
const hasBankFeeds = useFeatureGate("bank_feeds")
if (!hasBankFeeds) return <UpgradeBanner feature="Bank Feeds" />
```

### Feature Resolution

1. Load tenant's plan → get plan.features JSONB array
2. Load tenant's feature_override rows
3. Merge: override wins over plan default
4. Cache result per tenant (invalidate on plan/override change)

## Database Operations

### Backup

```bash
pg_dump -Fc --compress=9 -f /data/{slug}/backups/{timestamp}.dump {slug}
```

Compressed custom format. Stored in tenant directory.

### Restore

```bash
pg_restore -c -d {slug} /data/{slug}/backups/{timestamp}.dump
```

The `-c` flag drops existing objects before recreating.

### PITR (Point-in-Time Recovery)

Requires:
- WAL archiving enabled (`archive_mode = on`)
- WAL segments stored in `/data/wal-archive/`
- Recovery target specified by timestamp

### Google Drive Upload

Uses Google Drive API v3 with service account credentials:
1. Zip the backup file
2. Upload to configured Drive folder
3. Store Google Drive file ID in backup_history

# Risks / Trade-offs

- **Database creation permissions**: The API process needs
  `CREATEDB` privilege on PostgreSQL. In Fly.io this requires
  the superuser connection. May need a separate provisioning
  service or elevated credentials.
- **Flyway migration time**: Applying all migrations on a new
  database takes ~20-30 seconds. The UI must handle this
  gracefully (progress indicator, not a blocking request).
- **PITR complexity**: WAL archiving requires PostgreSQL
  configuration changes and significant storage. May defer to
  Phase 4+ depending on Fly.io Postgres capabilities.
- **Google Drive API**: Requires service account setup and
  OAuth consent. Deferred to Phase 4.
