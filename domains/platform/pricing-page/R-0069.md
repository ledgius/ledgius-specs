# Summary

The public pricing page at `ledgius.com/au/pricing` shall render
pricing cards dynamically from the platform database, supporting
rich markup content, country-specific pricing, competitive
positioning, and a clear path to signup. The existing inline
pricing section on the homepage is replaced with a link to this
dedicated page.

# Terms

| **Term** | Definition. |
|---|---|
| **Pricing region** | A country or market with its own currency, tax model, and competitive landscape. Initially `au` (Australia). Future: `nz`, `uk`, `us`. |
| **Rich content** | HTML-safe markup stored in the platform DB and rendered on the pricing page. Supports headings, bold, lists, links. NOT arbitrary HTML — sanitised subset only. |
| **Competitive context** | Positioning text that explains how a Ledgius plan compares to competitor pricing (Xero, MYOB, QuickBooks, Reckon). Stored per plan per region. |

# Problem / Business Context

The current pricing section is hardcoded in the homepage SPA with
static fallback data. It fetches from the API but the API returns
minimal data (name, price, feature list). This approach has several
problems:

1. **No country support** — pricing is AUD-only with no path to
   multi-country pricing.
2. **No rich content** — feature bullets are plain text. Cannot
   include formatting, links, or competitive context.
3. **No competitive positioning** — the pricing page needs to
   explain why Ledgius is better value than Xero/MYOB/QuickBooks
   at each tier.
4. **Inline on homepage** — pricing deserves its own page with
   room for detail, FAQs, comparison tables, and trust signals.
5. **Not admin-editable** — changes require code deploys.

Competitors have dedicated pricing pages with rich content:
- Xero: xero.com/au/pricing/
- MYOB: myob.com/au/accounting-software/pricing
- QuickBooks: quickbooks.intuit.com/au/pricing/

# Scope

## Pricing Page (PP-001 through PP-010)

- **PP-001**: A dedicated pricing page at `ledgius.com/{region}/pricing`
  (e.g., `/au/pricing`). The homepage pricing section is replaced
  with a "See Pricing" link to this page.
- **PP-002**: The page fetches pricing data from the public API
  endpoint `GET /api/v1/pricing?region={region}`. The API returns
  plans with rich content, feature lists, and competitive context.
- **PP-003**: The page renders pricing cards in a responsive grid
  (3-4 columns on desktop, stacked on mobile). Each card displays:
  - Plan name and tagline
  - Badge (e.g., "Most Popular", "Best Value")
  - Monthly and annual pricing with currency symbol
  - Monthly/annual toggle (shared across all cards)
  - Savings percentage for annual billing
  - Rich feature description (HTML-safe markup)
  - Feature bullet list with included/excluded indicators
  - CTA button ("Start Free Trial", "Get Started", "Contact Us")
  - Competitive context paragraph (optional, collapsible)
- **PP-004**: Below the pricing cards, the page shall include:
  - **Comparison table** — feature matrix across all plans
  - **Partner section** — accountant/bookkeeper console offering
  - **FAQ section** — common pricing questions
  - **Trust signals** — security, compliance, support guarantees

## Country/Region Support (PP-011 through PP-015)

- **PP-011**: Pricing is region-specific. Each plan has per-region
  pricing records with:
  - Region code (e.g., `au`, `nz`, `uk`, `us`)
  - Currency code and symbol (e.g., `AUD`, `A$`)
  - Monthly and yearly prices in local currency
  - Discount periods (region-specific promotions)
  - Tax note (e.g., "Prices include GST" for AU)
  - Competitive context (region-specific competitor comparison)
- **PP-012**: The URL structure is `/{region}/pricing`. The region
  is extracted from the URL path. Default: `au`.
- **PP-013**: The API returns plans filtered and priced for the
  requested region. Plans not available in a region are excluded.
- **PP-014**: The homepage "See Pricing" link uses the detected
  or default region: `ledgius.com/au/pricing`.
- **PP-015**: Future regions (NZ, UK, US) can be added by inserting
  pricing records in the platform DB — no code changes needed.

## Rich Content Model (PP-020 through PP-025)

- **PP-020**: Each plan's feature description supports a sanitised
  markup subset:
  - `**bold**` for emphasis
  - `- item` for bullet lists
  - `[link text](url)` for links
  - Line breaks preserved
  - NO arbitrary HTML tags, scripts, or iframes
- **PP-021**: The platform admin can edit plan content via the
  Pricing Plans page (R-0068 PA-030). The admin editor shows a
  preview of how the content renders on the public page.
- **PP-022**: Each plan carries these content fields per region:
  - `tagline` — one-line description (e.g., "For sole traders
    and owner-operators")
  - `feature_description` — rich markup paragraph describing
    what's included (e.g., "Unlimited quotes and invoices, bills,
    receipt capture, mileage, GST/BAS reporting...")
  - `feature_bullets` — JSON array of `{label, included}` items
    for the checklist display
  - `competitive_context` — rich markup paragraph explaining
    competitive positioning (e.g., "This sits below Xero Ignite,
    QuickBooks' A$33 entry point...")
  - `cta_text` — button label
  - `cta_url` — button destination (default: signup page)
- **PP-023**: Content is stored in a `pricing_plan_content` table
  keyed by `(plan_id, region)`. This separates content from
  commercial terms (pricing_plan_version).
- **PP-024**: The public API response merges plan identity +
  version pricing + region content into a single payload per plan.
  Internal IDs, feature gate identifiers, and entitlement keys
  are NOT exposed in the public response.
- **PP-025**: Content changes are immediately visible on the
  pricing page (no deploy needed). The API does not cache plan
  content — it reads from DB on each request.

## API Changes (PP-030 through PP-035)

- **PP-030**: Extend `GET /api/v1/pricing` to accept a `region`
  query parameter. Default: `au`.
- **PP-031**: Response shape per plan:
  ```json
  {
    "name": "Starter",
    "slug": "starter",
    "tagline": "For sole traders and owner-operators",
    "badge": "Best Value",
    "monthly_price": 19,
    "annual_price": 190,
    "currency": "AUD",
    "currency_symbol": "A$",
    "tax_note": "Prices include GST",
    "savings_pct": 17,
    "feature_description": "Unlimited quotes and invoices...",
    "feature_bullets": [
      {"label": "GST/BAS reporting", "included": true},
      {"label": "2 bank feeds", "included": true},
      ...
    ],
    "competitive_context": "This sits below Xero Ignite...",
    "cta_text": "Start Free Trial",
    "cta_url": "/signup",
    "sort_order": 1,
    "is_popular": false,
    "max_users": 1,
    "partner_section": null
  }
  ```
- **PP-032**: The partner plan (Ledgius Partner) is returned as a
  separate section with its own structure:
  ```json
  {
    "partner_plan": {
      "name": "Ledgius Partner",
      "tagline": "Free practice console for accountants and bookkeepers",
      "description": "Free practice console. Partner Compliance file at A$9/client/month...",
      "cta_text": "Join Partner Program",
      "cta_url": "/partner"
    }
  }
  ```
- **PP-033**: Admin endpoint `GET /platform/plans` continues to
  return the full plan data including internal fields. The public
  endpoint returns only display-safe fields.
- **PP-034**: The pricing API is public (unauthenticated) and
  should respond quickly (<200ms). No rate limiting needed for
  this endpoint.
- **PP-035**: CORS: the API must allow requests from
  `ledgius.com` and `www.ledgius.com` origins.

# Database Changes

## New table: `pricing_plan_content`

```sql
CREATE TABLE pricing_plan_content (
    id                  SERIAL PRIMARY KEY,
    plan_id             INT NOT NULL REFERENCES pricing_plans(id),
    region              TEXT NOT NULL DEFAULT 'au',
    currency            TEXT NOT NULL DEFAULT 'AUD',
    currency_symbol     TEXT NOT NULL DEFAULT 'A$',
    monthly_price       NUMERIC(10,2) NOT NULL,
    annual_price        NUMERIC(10,2) NOT NULL,
    tax_note            TEXT DEFAULT 'Prices include GST',
    tagline             TEXT,
    feature_description TEXT,
    feature_bullets     JSONB DEFAULT '[]',
    competitive_context TEXT,
    cta_text            TEXT DEFAULT 'Start Free Trial',
    cta_url             TEXT DEFAULT '/signup',
    is_active           BOOLEAN NOT NULL DEFAULT true,
    created_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
    UNIQUE (plan_id, region)
);
```

This separates regional content from the plan identity
(pricing_plans) and versioned commercial terms
(pricing_plan_version). A single plan can have content for
multiple regions with different currencies, prices, and
competitive positioning.
