# Summary

Manage a catalogue of products and services with sell/buy prices, tax code linkages, and GL account mappings. Products can be referenced from invoice and bill line items to auto-populate descriptions, prices, and account/tax settings.

# Problem / Business Context

Businesses sell products and services repeatedly. A product catalogue allows users to define items once with their default prices, tax codes, and GL accounts, then reference them when creating invoices or bills. This reduces data entry errors and ensures consistent pricing and account allocation.

# Scope

- Product CRUD (create, read, update, soft-delete via deactivation)
- Product types: product, service, overhead
- Sell and buy prices with default tax codes
- Income and expense account linkage
- SKU (optional, unique)
- Filtering by product type and search term
- Active/inactive status

# Out of Scope

- Inventory tracking (stock levels, warehouse management)
- Product images or media
- Product categories/groupings
- Pricing rules (volume discounts, customer-specific pricing)
- Product bundles/kits
- Barcode/QR code support

# Actors / Users

- Business Owner -- defines product catalogue
- Bookkeeper -- references products when creating invoices/bills
- Accountant -- reviews GL account mappings

# Preconditions

- User is authenticated
- Tax codes configured (for sell/buy tax code linkage)
- Income and expense accounts exist in chart of accounts

# Functional Requirements

- **PRD-001**: System shall store products with: SKU (unique, optional), name, description, product_type (product/service/overhead), unit, sell_price, buy_price, sell_tax_code_id, buy_tax_code_id, income_account_id, expense_account_id, and active flag
- **PRD-002**: Product type shall be validated to one of: product, service, overhead
- **PRD-003**: System shall support listing products with optional filters by product_type and search term (name/description), showing only active products by default
- **PRD-004**: System shall support partial update of product fields (name, description, unit, prices, tax codes, accounts, active status)
- **PRD-005**: Product deletion shall deactivate the product (soft delete), not physically remove it
- **PRD-006**: All monetary values (sell_price, buy_price) shall use shopspring/decimal
- **PRD-007**: Invoice and bill line items shall support an optional product_id reference

# Validation Rules

- Name is required
- Product type must be one of: product, service, overhead
- SKU must be unique if provided

# Error Handling

- Duplicate SKU returns constraint violation error
- Non-existent product ID returns 404
- Invalid product type returns validation error with allowed values
