# Summary

Create, view, and list accounts payable (AP) bills. Bill creation is a complete vertical slice that atomically creates an open item, transaction record, AP record, bill line items, and double-entry GL journal entries. Bills mirror the AR invoice pattern but with reversed debit/credit direction: credit AP (liability increases), debit expense accounts, debit GST Paid (asset increases).

# Problem / Business Context

Bills represent money owed to vendors. Ledgius must support creating bills with multiple line items, automatic GST calculation using configurable tax codes, and correct double-entry GL posting. The AP side mirrors the AR side but with reversed accounting entries.

# Scope

- Bill creation with multi-line items and automatic GL posting
- Bill listing with summary view (vendor name, amounts, status)
- Bill detail retrieval with lines and vendor name
- Vendor change on draft bills
- Per-line tax code support for configurable GST rates

# Out of Scope

- Draft bill update (line item replacement with GL recalculation)
- Bill PDF generation
- Bill approval workflow
- Purchase order matching
- Recurring bills (handled by R-0036)

# Actors / Users

- Business Owner -- reviews bills and approves payments
- Bookkeeper -- enters vendor bills into the system
- Accountant -- reviews expense categorisation

# Preconditions

- User is authenticated
- Vendor contact exists with a configured AP account (entity_credit_account)
- Expense accounts exist in the chart of accounts
- GST Paid account exists (if tax-inclusive lines)

# Functional Requirements

- **BILL-001**: System shall create an AP bill atomically: open_item + transaction + AP record + bill lines + acc_trans GL entries, all within a single database transaction
- **BILL-002**: GL entries shall follow double-entry rules: credit AP account (gross total, negative = liability increase), debit expense accounts (net per line, positive), debit GST Paid account (GST per line, positive = asset increase)
- **BILL-003**: GST shall be calculated per line item using the CEL rules engine, with rate looked up from tax_code if tax_code_id is provided, falling back to 10% default
- **BILL-004**: Default due date shall be calculated via CEL rule `default_due_date_days` from the transaction date if not explicitly provided
- **BILL-005**: System shall support changing the vendor on a draft bill (PATCH endpoint), rejecting changes on posted bills
- **BILL-006**: System shall list all bills with summary data: trans_id, invnumber, transdate, duedate, vendor_name, meta_number, amount_bc, curr, on_hold, approved
- **BILL-007**: System shall return bill detail with AP record, line items, and resolved vendor name
- **BILL-008**: Line items shall support optional product_id reference to the product catalogue

# Data / Entities / Fields

Primary entity: `APTransaction`

| Field | Type | Notes |
|-------|------|-------|
| trans_id | int (PK) | FK to transactions |
| invnumber | string (nullable) | Bill/invoice number |
| tax_included | bool | Whether prices include tax |
| duedate | date (nullable) | Payment due date |
| invoice | bool | Always true for bills |
| terms | int | Payment terms in days |
| notes | string (nullable) | |
| curr | string (nullable) | Currency code |
| ponumber | string (nullable) | Purchase order reference |
| on_hold | bool | Hold flag |
| entity_credit_account | int | FK to entity_credit_account (vendor) |
| is_return | bool | True for debit notes |
| amount_bc | decimal (nullable) | Gross total in base currency |
| netamount_bc | decimal (nullable) | Net total (excl GST) in base currency |
| open_item_id | int | FK to open_item |

Input: `BillLineInput`

| Field | Type | Notes |
|-------|------|-------|
| description | string | Required |
| qty | decimal | Required |
| sellprice | decimal | Required |
| account_id | int | Required, expense account |
| tax_account_id | int (nullable) | GST Paid account (legacy) |
| tax_code_id | int (nullable) | Configurable tax code for rate lookup |
| product_id | int (nullable) | Product catalogue reference |

# API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | /bills | List all bills (summary view) |
| GET | /bills/:id | Get bill with lines and vendor name |
| POST | /bills | Create a new bill |
| PATCH | /bills/:id/vendor | Change vendor on a draft bill |

# Validation Rules

- Vendor ID is required and must reference a valid entity_credit_account with an AP account
- Bill number is required
- Transaction date is required
- Currency is required
- At least one line item is required
- Each line requires description, qty, sellprice, and account_id

# Error Handling

- Vendor with no AP account returns error: "vendor N has no AP account configured"
- Vendor change on posted bill returns error: "cannot change vendor: bill is already posted"

# Acceptance Criteria

- [x] Bill creation atomically creates open_item, transaction, AP, lines, and GL entries
- [x] GL entries balance (credit AP = debit expense + debit GST)
- [x] Per-line tax code lookup from tax_code table
- [x] Default due date calculated via CEL rules engine
- [x] Vendor change on draft bill
- [x] Bill list with summary data
- [x] Bill detail with lines and vendor name

# Open Questions

1. **Draft bill update**: Unlike invoices, bills do not yet support full draft update with line replacement and GL recalculation.
2. **Bill PDF**: No PDF generation endpoint exists.
