# Summary

Generate standard financial reports: Trial Balance, Profit and Loss (P&L), Balance Sheet, AR Ageing, AP Ageing, GL Detail, Customer Statement, Vendor Statement, and Cash Flow Statement. Reports are read-only queries against the general ledger with correct accounting sign conventions.

# Problem / Business Context

Businesses require standard financial reports for management decision-making, compliance, and audit. Ledgius must generate these reports from the double-entry ledger data, correctly handling sign conventions (income stored as negative credits, displayed as positive), and enforcing the accounting equation (Assets = Liabilities + Equity).

# Scope

- Trial Balance (all accounts with non-zero balances, debit/credit columns)
- Profit and Loss (income and expenses for a date range, net profit)
- Balance Sheet (assets, liabilities, equity as at a date, with accounting equation verification)
- AR Ageing (customer receivables by age bucket: current, 30, 60, 90, 90+ days)
- AP Ageing (vendor payables by age bucket)
- GL Detail (transaction-level detail for a single account with running balance)
- Customer Statement (transaction history for a customer with running balance)
- Vendor Statement (transaction history for a vendor with running balance)
- Cash Flow Statement (operating, investing, financing sections with net change)

# Out of Scope

- Report export to CSV/Excel/PDF
- Comparative reports (prior period vs current)
- Budget-vs-actual reports
- Consolidated multi-entity reports
- Custom report builder
- Report scheduling and email delivery

# Actors / Users

- Business Owner -- reviews financial position
- Accountant -- generates reports for compliance and audit
- Bookkeeper -- reviews transaction detail and statements

# Preconditions

- User is authenticated
- Transactions exist in the ledger

# Functional Requirements

- **RPT-001**: Trial Balance shall return all accounts with non-zero balances, showing separate debit and credit columns
- **RPT-002**: Profit and Loss shall return income (category I) and expense (category E) accounts for a date range, with income amounts inverted from negative (ledger credits) to positive (display)
- **RPT-003**: Profit and Loss shall calculate net profit as total income minus total expenses
- **RPT-004**: Balance Sheet shall return assets (A), liabilities (L), and equity (Q) accounts as at a given date
- **RPT-005**: Balance Sheet shall invert liability and equity balances from negative (ledger credits) to positive (display)
- **RPT-006**: Balance Sheet shall verify the accounting equation (Assets = Liabilities + Equity) and return an error if it does not balance
- **RPT-007**: AR Ageing shall return customer receivables grouped by age buckets: current, 30 days, 60 days, 90 days, over 90 days, with totals per customer and grand totals
- **RPT-008**: AP Ageing shall return vendor payables grouped by the same age buckets
- **RPT-009**: GL Detail shall return transaction-level entries for a single account within a date range, including opening balance, line-by-line debit/credit/balance, and closing balance
- **RPT-010**: Customer Statement shall return transaction history (invoices, credit notes, receipts) for a customer with running balance
- **RPT-011**: Vendor Statement shall return transaction history (bills, debit notes, payments) for a vendor with running balance
- **RPT-012**: Cash Flow Statement shall return operating, investing, and financing sections with items, section totals, net change, opening cash, and closing cash

# Data / Entities / Fields

Key output structures:

`TrialBalanceLine`: accno, description, category, debit, credit

`ProfitLossReport`: from_date, to_date, income lines, expense lines, net_profit

`BalanceSheetReport`: as_at_date, assets, liabilities, equity, total_assets, total_liabilities, total_equity

`AgeingReport`: as_at_date, type (ar/ap), lines (contact with bucket breakdown), totals

`GLDetailReport`: account_id, accno, account_name, from_date, to_date, opening_balance, lines (entry-level detail), closing_balance

`StatementReport`: contact_id, contact_name, meta_number, contact_type, as_at_date, opening_balance, lines (transaction detail), closing_balance

`CashFlowReport`: from_date, to_date, operating/investing/financing sections, net_change, opening_cash, closing_cash

# Validation Rules

- Date parameters are required where specified (from_date, to_date, date)
- account_id required for GL detail
- customer_id required for customer statement
- vendor_id required for vendor statement

# Error Handling

- Missing required parameters return 400
- Balance sheet that does not balance returns the report with an error message indicating the imbalance amounts
