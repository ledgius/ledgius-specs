# Task Plan

## Phase 1: Foundation (DB + API + Data Sources)

### T-0033-01: Database migrations
**Estimate**: 2h
**Repo**: ledgius-db

- Create `report_templates` table (platform DB)
- Create `report_template_versions` table (platform DB)
- Create `report_generation_log` table (platform DB)
- Create `data_source_fields` table (tenant DB)
- Seed default field catalogues for built-in data sources

### T-0033-02: Data source registry
**Estimate**: 4h
**Repo**: ledgius-api

- Define `DataSource` interface (Slug, FieldCatalogue, Query)
- Define `DataField` struct
- Create `DataSourceRegistry` with Register/Get/List
- Implement data sources:
  - `profit_loss` — wraps existing ProfitLoss handler query
  - `balance_sheet` — wraps existing BalanceSheet handler query
  - `trial_balance` — wraps existing TrialBalance handler query
  - `invoice` — wraps existing GetInvoice with lines
  - `customer_statement` — wraps existing CustomerStatement query
  - `vendor_statement` — wraps existing VendorStatement query
- Register all sources in main.go at startup
- API: `GET /reports/data-sources` — list sources
- API: `GET /reports/data-sources/:slug/fields` — field catalogue

### T-0033-03: Template CRUD API
**Estimate**: 3h
**Repo**: ledgius-api

- Template repository (CRUD + versioning)
- Template handler (list, get, create, update, delete, clone)
- Register routes under `/reports/templates`
- Version creation on every update
- Tenant scoping (tenant_id filter)

### T-0033-04: HTML render engine
**Estimate**: 4h
**Repo**: ledgius-api

- Template JSON → HTML renderer
- Component renderers: text, data field, table, image, header,
  footer, page number, subtotal, spacer, horizontal rule
- Go html/template for hydration (data injection)
- Print CSS generation (page size, margins, breaks)
- API: `POST /reports/generate` (format: html|pdf)

## Phase 2: PDF Pipeline

### T-0033-05: Gotenberg deployment
**Estimate**: 2h
**Repo**: ledgius-infra

- Create `ledgius-gotenberg` Fly app
- Dockerfile / fly.toml for Gotenberg 8
- Deploy to Fly.io (syd region)
- Set `GOTENBERG_URL` secret on ledgius-api
- Verify HTML → PDF conversion works

### T-0033-06: PDF generation service
**Estimate**: 3h
**Repo**: ledgius-api

- `GotenbergPDFService` — POST HTML to Gotenberg, receive PDF
- `ChromedpPDFService` — fallback for local dev (no sidecar)
- Page settings: A4/Letter, portrait/landscape, margins
- Wire into report generation pipeline
- API: `POST /reports/generate` with format=pdf → returns PDF
- API: `GET /reports/generate/:id/download` for cached PDFs

## Phase 3: Template Editor (Frontend)

### T-0033-07: Install and configure Puck
**Estimate**: 2h
**Repo**: ledgius-web-app

- `npm install @measured/puck`
- Create `ReportDesignerPage` at `/reports/templates/:id/edit`
- Basic Puck editor rendering with default config
- Save/load JSON to/from API

### T-0033-08: Report components for Puck
**Estimate**: 6h
**Repo**: ledgius-web-app

- TextField component (static text, formatting)
- DataField component (bound to data source field)
- DataTable component (repeating rows, column config)
- Logo component (tenant logo or uploaded image)
- PageHeader / PageFooter components
- PageNumber component
- SubtotalRow component
- HorizontalRule / Spacer components
- Properties panel for each component

### T-0033-09: Data field picker
**Estimate**: 3h
**Repo**: ledgius-web-app

- Fetch field catalogue from API on data source selection
- Render fields grouped by category in a palette sidebar
- Drag fields from palette to canvas → creates DataField component
- Field type formatting (currency shows $, date shows DD/MM/YYYY)

### T-0033-10: Live preview
**Estimate**: 3h
**Repo**: ledgius-web-app

- Split editor: canvas (left) + preview (right) or toggle mode
- Preview calls `POST /reports/generate?format=html` with sample
  data and renders in an iframe
- Debounced re-render on template changes (500ms)
- Sample data provided by `GET /reports/data-sources/:slug/sample`

## Phase 4: Template Management UI

### T-0033-11: Template list page
**Estimate**: 2h
**Repo**: ledgius-web-app

- `ReportTemplatesPage` at `/reports/templates`
- List templates with: name, category, data source, last modified
- Create new / clone / delete actions
- Filter by category
- System templates shown as read-only with "Clone" action

### T-0033-12: Report generation page
**Estimate**: 3h
**Repo**: ledgius-web-app

- `GenerateReportPage` at `/reports/generate`
- Template selector
- Parameter form (date range, entity picker, filters)
- Preview button → HTML in iframe
- Download PDF button
- Print button → opens print dialog
- Generation history (recent reports)

## Phase 5: Print & Polish

### T-0033-13: Print optimisation
**Estimate**: 2h
**Repo**: ledgius-web-app + ledgius-api

- Print-specific CSS (page breaks, header repeat, margins)
- Print preview page (full-page HTML, no app chrome)
- `window.print()` integration
- Test across Chrome, Safari, Firefox

### T-0033-14: Default templates
**Estimate**: 4h
**Repo**: ledgius-db + ledgius-api

- Design and seed default templates:
  - Invoice (with lines, totals, payment details)
  - P&L (grouped by account category)
  - Balance Sheet (assets/liabilities/equity sections)
  - Customer Statement (transaction history, balances)
  - BAS Worksheet (GST summary fields)
- Seed as report_templates with is_default=true, tenant_id=NULL

### T-0033-15: Audit logging
**Estimate**: 1h
**Repo**: ledgius-api

- Log every report generation to `report_generation_log`
- Include: tenant, user, template, data source, params, format
- API: `GET /reports/history` — list recent generations

- Existing reporting queries (P&L, Balance Sheet, etc.) — already built

# Risks

| Risk | Mitigation |
|---|---|
| Puck may not support all report layout needs | Evaluate with a P&L template prototype before committing (T-0033-07) |
| Gotenberg cold start on Fly.io auto-stop | Set min_machines_running=1 or accept 2-3s first-request latency |
| Complex table layouts (nested groups, multi-level subtotals) | Start with flat tables, add grouping as a Phase 2 enhancement |
| PDF page breaks in mid-table | Use CSS `page-break-inside: avoid` on rows, test with large datasets |
